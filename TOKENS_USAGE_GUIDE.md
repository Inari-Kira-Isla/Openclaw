# Token 使用指南 v2

## 問題背景

⚠️ 當前問題：
- API rate limit 達到
- 多個 Agent 同時使用大量 tokens

## 解決方案：混合模型處理

### 策略流程

```
輸入 → 本地模型預處理 → 濃縮 → 雲端模型最終處理 → 輸出
         (Ollama)              (MiniMax)
```

### Token 節省

| 階段 | 模型 | Tokens 使用 |
|------|------|-------------|
| 原始 | 100% 雲端 | 100% |
| 優化 | 本地預處理 | 30% |

**節省：70%**

### 使用原則

| 任務類型 | 處理方式 |
|----------|----------|
| 簡單查詢 | 本地直接回覆 |
| 複雜任務 | 本地預處理 → 雲端最終 |
| 大量數據 | 分批本地處理 |

### Rate Limit 應對

1. 檢查剩餘配額
2. 本地模型優先
3. 不足則排隊等待
4. 切換備用模型

### 配額監控

- 每次大型操作前檢查配額
- 記錄使用量
- 預留 20% 配額
