# model-dispatcher Workflow

**Agent**: model-dispatcher
**功能**: 模型調度與選擇

---

## 工作流程

### Step 1: 接收任務

```
muse-core 傳遞任務
    ↓
分析任務內容
```

### Step 2: 評估維度

| 維度 | 評估 |
|------|------|
| 複雜度 | 簡單/中等/複雜 |
| 時效性 | 緊急/普通/不急 |
| 準確度 | 高/中/低 |
| 上下文 | 長/短 |

### Step 3: 模型選擇

```
評估結果
    │
    ├── 簡單 + 短上下文 → Ollama
    │
    ├── 中等 + 普通時效 → Ollama+
    │
    └── 複雜 + 高準確度 → MiniMax
```

### Step 4: 上下文打包

```json
{
  "system_prompt": "...",
  "relevant_memories": [...],
  "conversation_history": [...]
}
```

### Step 5: 執行與品質把關

```
執行結果
    │
    ├── 品質合格 → 回覆用戶
    │
    └── 品質不足 → 升級到 MiniMax
```

### Step 6: 反饋記錄

```
記錄：
- 選擇的模型
- 輸出品質
- 用戶評分
    │
    ↓
用於未來優化
```
