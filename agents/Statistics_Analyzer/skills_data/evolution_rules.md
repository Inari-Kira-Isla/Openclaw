# 技能進化觸發規則

## 概述
- **用途**: 根據使用情況自動觸發技能優化
- **建立日期**: 2026-02-17

---

## 進化觸發條件

### 1. 高頻使用觸發
```
當一個技能使用次數 > 100/月
→ 觸發「技能優化評估」
→ 檢查是否可以自動化更多流程
```

### 2. 低成功率觸發
```
當成功率 < 80%
→ 觸發「問題診斷」
→ 分析失敗原因
→ 優化提示詞
```

### 3. 低滿意度觸發
```
當平均評分 < 3.5
→ 觸發「改進計劃」
→ 收集用戶反饋
→ 重新設計技能
```

### 4. 等級晉升觸發
```
當經驗值達到閾值
→ 觸發「等級確認」
→ 記錄晉升
→ 慶祝 🎉
```

---

## 自動化進化流程

```
數據收集 → 分析 → 診斷 → 優化 → 驗證 → 記錄
```

### 每日 (23:00)
- 收集當日使用數據
- 更新經驗值
- 檢查觸發條件

### 每週 (週一 10:00)
- 效能分析
- 技能排名
- 優化建議

### 每月
- 全面回顧
- 等級評估
- 系統升級

---

## 等級閾值

| 等級 | 經驗值 | 權限 |
|------|--------|------|
| Novice | 0-100 | 基礎功能 |
| Intermediate | 101-500 | 進階功能 |
| Advanced | 501-1000 | 完整功能 |
| Expert | 1001-2000 | 自訂功能 |
| Master | 2001+ | 系統優化 |

---

## 優先順序

1. **高頻使用技能** → 優先優化
2. **低成功率技能** → 優先診斷
3. **新技能** → 觀察期
