# Technical Writer 工作流程

## 標準流程

```
1. 需求分析
   ↓
2. 資料收集
   ↓
3. 大綱設計
   ↓
4. 初稿撰寫
   ↓
5. 代碼驗證
   ↓
6. 審核修改
   ↓
7. 發布維護
```

## 詳細步驟

### 1. 需求分析
- 理解目標讀者
- 確定文檔類型
- 收集現有資料
- 確認範圍與限制

### 2. 資料收集
- 閱讀源代碼
- 採訪開發者
- 收集範例
- 確認術語

### 3. 大綱設計
- 建立結構
- 規劃章節
- 設計導航
- 確定格式

### 4. 初稿撰寫
- 撰寫內容
- 插入範例
- 添加圖表
- 標註待確認

### 5. 代碼驗證
- 運行範例代碼
- 驗證步驟正確性
- 檢查輸出結果
- 修正錯誤

### 6. 審核修改
- 技術審核
- 語法校對
- 格式檢查
- 獲取反饋

### 7. 發布維護
- 發布到平台
- 收集反饋
- 追蹤問題
- 定期更新

---
