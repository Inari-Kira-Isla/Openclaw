# Agent Builder Workflow

## 建構流程

### 第一階段：需求分析

#### 1.1 了解目標
- 與用戶溝通
- 確定 Agent 用途
- 了解目標用戶

#### 1.2 分析任務
- 識別任務類型
- 評估複雜度
- 確定範圍

#### 1.3 收集需求
- 功能需求
- 非功能需求
- 約束條件

### 第二階段：身份設計

#### 2.1 命名
- 選擇名稱
- 設定頭銜
- 定義角色

#### 2.2 性格塑造
- 核心特質
- 溝通風格
- 專業態度

#### 2.3 專業領域
- 主要擅長
- 次要擅長
- 持續學習方向

### 第三階段：工作流程設計

#### 3.1 任務接收
- 接收任務
- 理解需求
- 確認理解

#### 3.2 執行規劃
- 拆分任務
- 規劃步驟
- 設定順序

#### 3.3 執行監控
- 執行任務
- 追蹤進度
- 處理問題

#### 3.4 完成輸出
- 整理結果
- 質量檢查
- 交付成果

### 第四階段：技能清單

#### 4.1 核心技能
- 主要能力
- 專業技能
- 技術能力

#### 4.2 輔助技能
- 溝通能力
- 分析能力
- 學習能力

#### 4.3 專業領域
- 細分領域
- 子領域
- 專業深度

### 第五階段：文檔創建

#### 5.1 結構設計
- 文件框架
- 內容大綱
- 格式規範

#### 5.2 內容撰寫
- agent.md
- workflow.md
- soul.md
- skill.md

#### 5.3 測試驗證
- 功能測試
- 流程驗證
- 效果評估

### 第六階段：優化迭代

#### 6.1 收集反饋
- 用戶反饋
- 性能數據
- 使用情況

#### 6.2 迭代改進
- 優化流程
- 增強技能
- 完善設定

#### 6.3 持續學習
- 學習新技術
- 追蹤趨勢
- 改進方法

## 輸出格式

```
## 需求分析
## 身份設計
## 工作流程
## 技能清單
## 文檔結構
## 測試結果
```
