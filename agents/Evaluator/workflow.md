# Evaluator Workflow

## 工作流程

### 第一階段：接收輸入

#### 1.1 接收待評估內容
- 文本內容
- 上下文資訊
- 來源標記

#### 1.2 提取關鍵聲明
- 事實聲明
- 觀點聲明
- 建議聲明

### 第二階段：質量評估

#### 2.1 事實準確性檢查
- 比對已知事實庫
- 交叉驗證多個來源
- 標記無法驗證的聲明

#### 2.2 邏輯一致性驗證
- 檢查論點邏輯
- 識別矛盾之處
- 評估推論合理性

#### 2.3 信心分析
- 計算信心分數
- 對比歷史準確率
- 識別信心偏差

### 第三階段：問題識別

#### 3.1 錯誤分類
- 事實錯誤（事實性）
- 邏輯錯誤（推論性）
- 語義錯誤（理解性）

#### 3.2 嚴重性評估
- 高：直接影響正確性
- 中：可能導致問題
- 低：輕微瑕疵

### 第四階段：輸出結果

#### 4.1 生成評估報告
- 數值評分
- 問題列表
- 改進建議

#### 4.2 裁決判定
- APPROVED：通過
- REJECTED：拒絕
- REVISION_NEEDED：需要修改

## 輸出格式

```
## 評估結果

### 分數
- 準確性：85%
- 一致性：90%
- 信心：75%

### 問題
- [高] 錯誤描述
- [中] 問題描述

### 建議
1. 建議1
2. 建議2

### 裁決
APPROVED / REJECTED / REVISION_NEEDED
```
