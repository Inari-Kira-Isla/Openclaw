---

## 核心觀點

> **AI Agent 正在從「介面」變成「基礎設施」**

---

## 六大發展路線

### 1. 瀏覽器/IDE → Agent 身體

| 發展 | 說明 |
|------|------|
| Google Chrome | Auto Browse 功能，Gemini 直接執行多步驟網頁任務 |
| Vercel | agent-browser 支援 iOS |
| Cloudflare | agent + workflow 整合 |
| Apple Xcode | Claude Agent SDK 整合 |

### 2. Agent 管理平台 (Agent OS 雛形)

| 產品 | 功能 |
|------|------|
| **OpenAI Frontier** | shared memory, onboarding, permissions, feedback loop, deployment |
| Cloudflare | workflow + agent 整合 |

> **Agent 變成可管理的數位同事系統**

### 3. 多 Agent 協作產品化

| 產品 | 說明 |
|------|------|
| Perplexity Model Council | 多個 frontier model 同時回答，做 consensus |
| Firecrawl | 支援 parallel agents，同時跑數千個 agent 查詢網頁 |

> **從概念變成 infrastructure**

### 4. Coding Agent 標準化與反思

| 發展 | 說明 |
|------|------|
| NVIDIA | 使用 Cursor 程式碼產量 3 倍，bug rate 沒上升 |
| Cursor | Agent Trace open spec，處理 agent 產生的可追溯性 |
| Claude Code | /insights 分析使用紀錄，提出 workflow 建議 |

> **Coding agent 已成長期工具**

### 5. 研究/寫作專用工作空間

| 產品 | 說明 |
|------|------|
| **Prism** | LaTeX 原生雲端工作區，GPT 5.2 整合進專案 |
| Perplexity Deep Research | 升級，DRACO benchmark 強化 |

> **從高密度文字和邏輯場景切入**

### 6. 自主 Agent + 本地部署

| 發展 | 說明 |
|------|------|
| Claude Cowork | 長期存在 |
| 龍蝦框架 | 自主運作 |
| Codex App | 自動化 |
| Helius | 支援 agent 自動生成 API keys |

> **讓 AI 從需要呼叫變成可持續存在**
