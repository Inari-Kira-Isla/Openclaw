# AI 個人化寫作系統 - Wilson Huang 方法論

**日期**: 2026-02-20
**來源**: Wilson Huang (Medium)
**標籤**: #AI寫作 #個人化 #PromptEngineering

---

## 核心問題

AI 寫出既野「一眼就睇得出黎」：
- 太工整
- 每段一樣長
- 鍾意用「首先⋯⋯其次⋯⋯最後⋯⋯」
- 「值得注意的是」開頭
- 「不是 A 而是 B」句型

---

## 解決方案：4步流程

### Step 1: 俾 AI 讀晒你既舊文章

| 平台 | 代表 |
|------|------|
| Medium | 認真寫既模式 |
| Facebook | 隨手寫既模式 |

點解要兩個？
- Medium → 有結構、有鋪排
- Facebook → 口語、即興

**建議素材**：
- 5-10 篇滿意既長文
- 20-30 則社群短文
- 時間跨度越長越好

---

### Step 2: 叫 AI 分析寫作風格

三個維度：

| 維度 | 問題 |
|------|------|
| 口吻 | 正式定口語？有咩幽默模式？ |
| 寫作方式 | 結構點樣？開頭結尾慣例？ |
| 用字習慣 | 常用詞彙？口頭禪？ |

**有趣發現既例子**：
- 超鍾意用 TL;DR 開頭
- 中英文混用有隱性規則
- 一堆口頭禪自己唔知：「不外乎」、「根本就是」、「說白了」
- 自嘲頻率比想像中高

---

### Step 3: 把分析提煉成 System Prompt

六個區塊：

#### 區塊一：身份背景
- 你係邊個、做咩既
- 咩影響用詞同視角

#### 區塊二：語氣規則
```
Do：用「我」開頭，可以穿插「老實說」、「說白了」
Don't：不要用「筆者」、「本文」、「值得注意的是」
```

#### 區塊三：用字清單
- 會用既詞（口頭禪清單）
- 唔會用既詞（禁忌清單）

**禁忌清單好重要！**
- AI 既壞習慣：「深入探討」、「全面解析」、「不僅⋯⋯更⋯⋯」
- 直接寫死：「絕對唔好用呢個句型」

#### 區塊四：文章架構模板
- 長文：TL;DR → 背景 → 分段論述 → 前瞻結尾
- 短文：一句话觀察 → 分享+短評 → 反思

#### 區塊五：核心信念
- 價值觀會影響論述方向
- 例如：「獨立思考比追隨群眾重要」

#### 區塊六：寫作禁忌
- 10 條「Anti-AI 味」清單
- 段落長短要有變化
- 唔好過度使用轉折詞
- 唔好用對稱排比句

---

### Step 4: 放入 Claude 專案迭代

- 建一个新專案
- 把 Prompt 貼入 Project Instructions
- 每次寫得「唔對」既時候
- 諗「係邊度唔對？」
- 把問題加返入 System Prompt

**迭代例子**：
- 問題：段落太工整，全部一樣長
- 修正：「段落長短要有變化，有時一句話、有時五六句」

## 幾個有趣結論

### 1. 寫作風格比想像中穩定
- 七八年間「講野既方式」幾乎冇變
- 變既係主題同深度

### 2. 最好既 Prompt 係禁止清單
- 正向指令效果唔好：「請用輕鬆既語氣」
- 負向指令勁好多：「唔好用以下呢啲詞」

### 3. 社群短文更代表你
- 長文經過修改，已經唔夠「真實」
- Facebook / Twitter 短文更接近「原始說話方式」

### 4. 「似你自己」係可以量化既
- 用咩詞、唔用咩詞
- 段落幾長
- 比喻幾多
- 自嘲幾多
- 中英文點樣混合
- 結尾習慣
