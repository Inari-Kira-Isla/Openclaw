## 🎯 團隊共識

### 核心共識：系統的挑戰不在「儲存」，而在「什麼時候忘記」

### 三層架構

```
┌─────────────────────────────────────────────────────────┐
│                    感知層 (Perception)                   │
│    資料攝入、特徵提取、初步清洗                           │
│              ↓ 事件驅動                                   │
├─────────────────────────────────────────────────────────┤
│                    整合層 (Integration)                 │
│    記憶編碼、策略生成、模式識別                           │
│    [記憶層] + [學習機制] 協作                            │
│              ↓ 行動觸發                                   │
├─────────────────────────────────────────────────────────┤
│                    輸出層 (Output)                        │
│    執行記憶強化、干預建議、反饋閉環                       │
└─────────────────────────────────────────────────────────┘
```

**協作關鍵**：事件匯流排串連三層，資訊雙向流動（非單向）。

## 📊 成功標準

### 量化指標
- 記憶召回命中率 > 85%
- 檢索延遲 < 50ms (P95) / < 200ms (P99)
- 學習任務完成效率提升 ≥ 15%
- 系統可用性 ≥ 99.5%

### 質化指標
- 使用者主觀感受到「史萊姆真的記得之前說過什麼」
- 干預建議採納率 > 40%

### 風險閾值
- 任何資料洩露事故 = 失敗
- 用戶體驗嚴重下降連續 3 天 = 緊急回滾
