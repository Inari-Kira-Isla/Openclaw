# 三層 Email 分類 Workflow

**日期**: 2026-02-20

---

## 三層過濾流程

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: Gmail Filter (免費)                           │
│  - Promotions / Social / Important / Forums           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Layer 2: 本地模型 (deepseek-coder:1.3b)                │
│  - 快速分類: 重要 / 一般 / 待辦 / 垃圾                  │
│  - 776MB, 載入快                                       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: OpenClaw (複雜郵件)                           │
│  - 需要理解的郵件                                       │
│  - 需要回覆的郵件                                       │
│  - 需要執行動作的郵件                                   │
└─────────────────────────────────────────────────────────────┘
```

---

## n8n Workflow 設計

### Layer 1: Gmail 取得郵件
```
節點: Gmail - Get Emails
過濾: is:unread label:inbox
數量: 每次 50 封
```

### Layer 2: 本地模型分類
```
節點: Code - 本地分類
模型: deepseek-coder:1.3b
邏輯:
  if "促銷/社交/論壇" → Gmail Filter 已處理
  if "簡單分類" → 加標籤 → 完成
  if "複雜" → Layer 3
```

### Layer 3: OpenClaw 處理
```
節點: HTTP Request → OpenClaw API
條件: 複雜郵件
```
