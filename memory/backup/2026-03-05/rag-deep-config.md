# RAG 深度關聯配置

## 核心原則
- **向量延伸**：不只是關鍵詞，而是概念擴展
- **深度挖掘**：理解上下文，而不僅僅匹配
- **關係建立**：向量之間的關聯比數量更重要

## 優化配置

### 檢索參數
| 參數 | 舊值 | 新值 | 說明 |
|------|------|------|------|
| 相似度閾值 | 0.7 | 0.5 | 更深入的匹配 |
| Top-K | 5 | 10 | 更多相關結果 |
| 關聯擴展 | 關閉 | 開啟 | 擴展查詢範圍 |

### 關係圖譜
```
Kira → 決策 → 發起任務
Neicheok → 裁決 → 審視決定
Evolution → 質疑 → 檢視改進
Cynthia → 知識 → 管理記憶
Slime → 學習 → 持續優化
Team → 執行 → 落實任務
```

### 向量類型
1. **概念向量**：核心概念的多維度表達
2. **上下文向量**：環境/背景相關
3. **關係向量**：與其他向量的連接
