## 核心問題

> Prompt 工程能優化回答品質，卻難以保證做事的穩定度。

LLM 兩大痛點：
1. **缺乏持久情境** - 每次都要重新交代 SOP
2. **缺乏程序化知識** - 說得一口好菜卻做不出菜

---

## Skills 的核心價值

### 封裝 (Encapsulation)

將 Prompt、規則、SOP 與模板打包成可重複載入的能力模組。

| 價值 | 說明 |
|------|------|
| 領域專業化 | 將行業 Know-how 固化進 Agent |
| 流程穩定化 | 在非決定性系統中建立可預測路徑 |

---

## 漸進式揭露 (Progressive Disclosure)

像人類專家的「索引式大腦」- 平常不佔用空間，需要時才調用。

### 三層加載機制

| 層級 | 時機 | 內容 |
|------|------|------|
| 1. 宣告 | 始終加載 | 名稱與描述 |
| 2. 引導 | 觸發時 | SKILL (SOP 說明書) |
| 3. 操作 | 執行時 | 腳本與參考文件 |

---

## Agent 生態四位一體

```
┌─────────────────────────────────────┐
│         Agent (編排工程)            │
├─────────────────────────────────────┤
│  MCP (Data Layer)    - 資料供應    │
│  Tools (Execution)   - 執行指令    │
│  Skills (Logic)     - 流程與知識   │
│  Subagents (Modular) - 隔離 Context │
└─────────────────────────────────────┘
```

## 工程實踐

### 檔案結構
```
SKILL.md          # 能力說明
metadata.yaml     # 元數據
```

### 關鍵指標

1. **語意路由** - 描述「何時使用 (When)」而非「做什麼 (What)」
2. **動能一致性** - 使用動名詞 (ing) 命名，強調行為屬性
3. **元能力化** - 透過 Skill Creator 自動生成標準化模組

---

## 洞察：Skill 將知識資產化

> 當企業把經營 Know-how 封裝成 SKILL，這個「能力」就變成了公司的數位資產。

- 不休息
- 不離職
- 能同時驅動多個 Agentic AI
