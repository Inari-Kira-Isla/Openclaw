# Email 分類方案分析

**日期**: 2026-02-20
**問題**: 1000+  emails token 消耗大、重複性工作

---

## 方案比較

### 方案 1: 全部本地模型

| 項目 | 說明 |
|------|------|
| **優點** | 無 API 成本、隱私保護 |
| **缺點** | 模型載入慢 (30-60秒)、分類能力有限 |
| **適用** | 少量郵件測試 |

### 方案 2: Gmail 預篩選 + 本地模型

| 項目 | 說明 |
|------|------|
| **優點** | 免費、快速、實用 |
| **缺點** | 需要設定規則 |
| **適用** | 大量郵件 |

---

## 推薦方案：三層過濾

```
Gmail Filter (免費)
    ↓
本地模型分類 (快速)
    ↓
OpenClaw/MiniMax (複雜才用)
```

### 第一層：Gmail 預設分類

使用 Gmail 內建：
- 促銷 (Promotions)
- 社交 (Social)
- 重要 (Important)
- 論壇 (Forums)

### 第二層：本地模型 (deepseek-coder:1.3b)

```python
# 快速分類
model = "deepseek-coder:1.3b"  # 776MB, 最快
prompt = "分類: 重要/一般/垃圾"
```

### 第三層：OpenClaw (複雜郵件)

- 需要理解的郵件
- 需要回覆的郵件
- 需要執行動作的郵件

---

## 實際流程

1. **Gmail 取得未讀郵件** (最多 500封/次)
2. **本地模型快速分類** → 加標籤
3. **複雜郵件才用 OpenClaw**

---

## Token 節省估算

| 方案 | 1000 封郵件 |
|------|-------------|
| 全部 OpenClaw | ~500,000 tokens |
| Gmail Filter + 本地 | ~50,000 tokens (90% 節省) |
| 三層過濾 | ~10,000 tokens (98% 節省) |

---

## 建議行動

1. 先用 Gmail 內建過濾
2. 用本地模型處理簡單分類
3. 只對重要郵件用 OpenClaw
