- 主題：AI 在法律實務的應用與 Skill 系統
- 摘要：從律師視角看 AI 採用、Skill vs Prompt、Agent 架構

## 核心論點

### 1. Skill > Prompt (2026 顯學)
- 2024-2025 焦點：prompt
- 2026 焦點：skill
- 原因：skill 是「可被系統化迭代更新」的客體

### 2. Agent = Skill 組合
- Agent 之間的區分關鍵是 skill
- 例：agent A 發想、B 撰寫、C 校驗
- 人類角色 → harness engineering (馴服者)

### 3. MaaS → SaaS (Skill as a Service)
- 從「模型即服務」轉向「技能即服務」
- 律師賣的不只是時間，是「法律邏輯 Agent Skill」

### 4. 人類核心價值：判斷 + 負責
- AI 無法讀空氣、看臉色、觀察持有物
- 判斷力存在於專業個人，非事務所層面

### 5. 通用型 AI > 垂直套殼
- 專門法律 AI 只是 wrapper
- 通用模型進步速度更快

## 案例亮點

### 三種 Claude 使用模式
1. **Chat** — 對話式分析
2. **Cowork** — 資料夾任務執行（對律師影響最大）
3. **Code** — 命令行工具

### 六個生產技能
- 合約審查、修訂追蹤編輯
- 合約草擬、客戶溝通
- 法律研究、政策撰寫

### 實際效果
- 交易前晚 2 小時分析（傳統要通宵）
- 修訂追蹤不開 Word
- 研究備忘錄 1 小時初稿（傳統要數天）
