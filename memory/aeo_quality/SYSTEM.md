# AEO 質量保障系統

## 系統架構

```
┌─────────────────────────────────────────────────────────────┐
│                    AEO 內容閉環系統                        │
└──────────────────────┬────────────────────────────────────┘
                       │
       ┌───────────────┼───────────────┐
       ▼               ▼               ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 趨勢收集  │    │ 文章生成 │    │ 質量檢查 │
└──────────┘    └──────────┘    └──────────┘
                              │
                              ▼
                    ┌──────────────┐
                    │ 質量閾值    │
                    │ ≥ 70 分    │
                    └──────────────┘
```

## 質量標準

### 字數要求
- 最低：1000 字
- 建議：1500 字

### 結構要求
- H1：1 個（標題）
- H2：至少 3 個
- H3：至少 2 個

### 代碼範例
- 至少 1 個
- 語言：Python, JavaScript, Bash, YAML, JSON

### SEO 要求
- title：必備
- description：必備
- keywords：至少 3 個
- JSON-LD：必備

## 評分權重

| 項目 | 權重 |
|------|------|
| 字數 | 20% |
| 結構 | 30% |
| 代碼範例 | 20% |
| SEO | 20% |
| 內容深度 | 10% |

## 閾值

| 分數 | 動作 |
|------|------|
| ≥ 90 | 通過，Excellent |
| 70-89 | 通過，Good |
| < 70 | 阻止發布 |

## 工作流

1. **趨勢收集** → 2. **文章生成** → 3. **質量檢查** → 4. **部署發布**
