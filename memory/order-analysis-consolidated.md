## 1. 需求與功能定義

### 功能需求（優先級）

| # | 功能 | 優先級 | 說明 |
|---|------|--------|------|
| 1 | 訂單提取 | P0 | 從 WeChat 對話中識別並提取訂單 |
| 2 | 商品分類 | P0 | 按魚類/貝類/蝦類分類 |
| 3 | 客戶識別 | P0 | 識別餐廳/客戶名稱 |
| 4 | 價格計算 | P1 | 計算訂單總額 |
| 5 | 數據統計 | P1 | 銷售排行、客戶統計 |
| 6 | 報表生成 | P2 | 生成每日/每週報告 |

### 輸入格式

```
[客戶名] [商品] [數量] [價格]
```

來源: WeChat 群組訂單訊息

### 輸出格式 (JSON)

```json
{
  "orders": [
    {
      "customer": "客戶名",
      "items": [
        {"name": "商品名", "qty": 數量, "unit": "單位", "price": 價格}
      ],
      "total": 總額
    }
  ],
  "summary": {
    "total_orders": 數量,
    "total_revenue": 金額,
    "top_products": ["熱門商品"],
    "customers": ["客戶列表"]
  }
}
```

### 評估指標

| 指標 | 目標 |
|------|------|
| 訂單識別率 | >95% |
| 商品分類準確率 | >90% |
| 價格計算準確率 | >98% |
| 客戶識別準確率 | >95% |

## 4. 優化後 Prompt

```markdown
你是一個專業的日本料理訂單分析助手。

## 輸入
WeChat 訂單訊息

## 輸出格式 (JSON)
{
  "customer": "客戶名",
  "items": [{"name": "商品", "qty": 數量, "unit": "單位", "price": 價格}],
  "total": 總額
}

## 規則
1. 價格如有"機價"或其他備註，設為 null
2. 數量單位: 條/k/kg/箱/盒/包
3. 客戶名: 餐廳名/酒店名/地名
```

### Prompt 優化方向

- 客戶名稱識別模式
- 商品數量提取規則
- 價格計算邏輯
- 邊界情況處理（如「機價」、「加單」）
- 標準化 JSON Schema 及欄位驗證
