# Web Search 有效使用反饋系統

**建立日期**: 2026-02-19
**目標**: 最大化每 次搜尋的價值

---

## 🎯 核心原則

### 有效使用 = 有行動 + 有結果

| 標準 | 定義 |
|------|------|
| **高價值** | 搜尋 → 知識庫更新 → 系統優化 |
| **中等價值** | 搜尋 → 記錄但未應用 |
| **低價值** | 搜尋 → 無法使用/重複 |

---

## 📊 反饋循環

```
搜尋 → 分析 → 評估 → 應用 → 記錄 → 優化
  ↑                                    ↓
  └────────── 每週總結反饋 ←────────────┘
```

---

## 🎯 搜尋分配策略

### 每日 20 次分配

| 類別 | 次數 | 目標 | 反饋 |
|------|------|------|------|
| **趨勢** | 5 | 新聞/趨勢 | 知識庫更新? |
| **技術** | 5 | 工具/更新 | 系統優化? |
| **應用** | 5 | 實作案例 | 可執行? |
| **分析** | 3 | 競品/市場 | 有洞見? |
| **保留** | 2 | 緊急 | 需要? |

---

## 📈 評估矩陣

### 每次搜尋後評估

| 問題 | 評分 |
|------|------|
| 1. 有新知識嗎？ | 1-5 |
| 2. 可以應用嗎？ | 1-5 |
| 3. 值得記錄嗎？ | 1-5 |
| **總分** | **/15** |

### 分類
- **12-15**: 高價值 → 保持此類主題
- **8-11**: 中價值 → 優化關鍵詞
- **<8**: 低價值 → 避免/調整
