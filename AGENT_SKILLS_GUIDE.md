# Agent Skills 使用指南：讓 AI 記住你的工作方式

> 篇者：Joe Cheok | 編輯：Claude

---

## 從雇一個助理說起

想像你雇了一個新助理，第一週每件事都要手把手教，格式、邏輯、注意事項逐一說明。如果你能把這些全部寫成一份說明書，下一個助理直接照做，不用重教——**Agent Skill 就是那份說明書，只是對象換成了 AI。**

更具體一點，想像你去星巴克點咖啡，如果每次都要從頭說一遍：「我要大杯、燕麥奶、少糖、不加糖漿，冰的、拿鐵」，點個咖啡要花三十秒，店員也容易聽錯。但如果你是常客，跟店員說一句「老樣子」，對方就知道要幫你做什麼了。

**Skill 就是那句「老樣子」**。它是你幫 AI 預設好的一套「遇到這類任務，你該這樣做」的指令包。

---

## 這跟我一般跟 AI 對話有什麼差？

大多數人使用 ChatGPT 或 Claude 網頁版的方式大概是這樣：打開新的對話視窗，重新說一遍「請幫我用這個格式整理以下內容……」，然後把資料貼上去。

這沒什麼問題，但有幾個成本在裡面：
- 每次要記得那段指令長什麼樣子
- 換個人用就要重新交代一遍
- 指令一長就很難維護
- 上次花十分鐘調好的輸出格式，下次開新對話又得從頭來

**Skill 做的事，就是把那段你每次都要重複說的話，變成一個可以重複呼叫的標準作業流程。**

---

## 跟把指令存在記事本有什麼不一樣？

概念上類似，但 Skill 多做了幾件事：

| 特性 | 記事本 | Skill |
|------|--------|-------|
| 自動觸發 | ❌ 需手動貼上 | ✅ AI 自動識別 |
| 結構化 | ❌ 純文字 | ✅ 定義情境/工具/步驟 |
| 共用 | ❌ 難分享 | ✅ 一人建立，全團隊使用 |

---

## 跟 GPT / Gem 有什麼不同？

GPT 和 Gem 每個是獨立的對話入口，你要用哪個功能就要點進那個對話。

**Skill 的運作邏輯不一樣**：你在同一個對話裡，可以依照任務需要，隨時呼叫不同的 skill，甚至在一個任務裡同時用到好幾個。

> 舉例：你說「幫我整理這份會議記錄，然後根據結論草擬 email」。AI 可以先呼叫「會議紀錄整理 skill」，整理完後接著呼叫「客戶 email 撰寫 skill」，一氣呵成。

## 一個 Skill 長什麼樣子？

每個 skill 的核心，是一份純文字的 Markdown 檔案（SKILL.md），回答三個核心問題：

### 1. 觸發時機
這個 skill 叫什麼名字、什麼時候出現？—— 決定 AI 在哪些情況下會用它

### 2. 可用工具
它能用哪些工具？—— 有些 skill 只讀文字就夠了，有些需要連到 Google Drive、查網路

### 3. 執行流程
按哪個順序做事、要注意什麼、輸出格式長什麼樣？—— 這就是你的 SOP

```
weekly-report-skill/
  SKILL.md          ← 主設定
  reference.md      ← 補充參考（可選）
  template/         ← 輸出模板（可選）
  scripts/          ← 程式碼（可選）
```

## MCP 是什麼？跟 Skill 有什麼關係？

| 角色 | 職責 |
|------|------|
| **MCP** | 「AI 能不能拿到那個資料」— 外部工具接口 |
| **Skill** | 「拿到資料之後該怎麼處理」— 處理流程 |

MCP = 進貨通道  
Skill = 廚師食譜

分工很清楚：MCP 解決「能不能拿」，Skill 解決「怎麼處理」。

---

## 怎麼建一個 Skill？

兩條路，不需要會寫程式：

### 1. 用 Skill Creator
Skill Creator 本身就是一個 skill，是「幫你建 skill 的 skill」。描述你想做的事，它幫你生出 SKILL.md。

### 2. 直接跟 Claude 對話打包
把工作流程用白話說給 Claude 聽，讓它幫你整理成 SKILL.md 檔案。

---

## Skill 的限制

| 限制 | 說明 |
|------|------|
| 維護成本 | 流程改了，skill 也要跟著更新 |
| 記憶問題 | 每次新對話，AI 不會記得你上次說了什麼 |
| 打架問題 | 多個 skill 觸發條件重疊時，可能選錯 |

**解決方式**：讓觸發說明具體有區別性，避免描述太泛。

## 哪些平台支援？

| 類別 | 平台 |
|------|------|
| AI 對話 | Claude Chat, Claude Code, Gemini, ChatGPT |
| 開發工具 | Cursor, Windsurf |
| 企業工具 | Notion, Figma, Jira, Confluence, Canva |
| 自動化 | Zapier, Stripe, Manus |

**重要的是**：你建的 SKILL.md 格式相同，跨工具都能用。

## 從哪裡開始？

想想你這個月重複做了三次以上的事，有沒有每次都要重新跟 AI 解釋格式或流程的？

**那就是最值得先建成 skill 的任務。**
