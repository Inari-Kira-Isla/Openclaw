# 工作流協調

規劃並執行多步驟工作流，管理步驟依賴、並行執行與進度追蹤。

## 操作 / 工作流程

1. **任務拆解** — 將複雜任務分解為最小顆粒度步驟（單一職責、明確輸入輸出）
2. **依賴分析** — 識別步驟間的依賴關係，決定執行順序
3. **流程編排** — 選擇流程模式：
   - **線性**：A → B → C（每步依賴前一步）
   - **並行**：A + B 同時執行 → C（無依賴可並行）
   - **條件**：A → 判斷 → B 或 C（依條件分支）
4. **執行監控** — 使用 `state_control` 追蹤進度，錯誤隔離不影響無依賴步驟
5. **結果彙整** — 收集各步驟輸出，交由 `result_integrator` 整合

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| task | string | — | 任務描述 |
| mode | string | auto | 流程模式：linear / parallel / conditional / auto |
| max_steps | number | 10 | 最大步驟數 |
| timeout_per_step | number | 60 | 每步驟超時秒數 |

## 輸出格式

```
🎯 工作流：[任務名稱]
狀態：[running / completed / failed]
進度：[2/5] ████░░░░░░ 40%

步驟：
✅ 1. [步驟一] — 完成
⏳ 2. [步驟二] — 執行中
⬚  3. [步驟三] — 等待中
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 步驟執行超時 | 標記該步驟失敗，跳過無依賴的後續步驟繼續執行 |
| 依賴步驟失敗 | 標記所有依賴鏈步驟為 skipped |
| 所有步驟失敗 | 彙整錯誤報告，交由 failure_analysis 分析 |
| 任務無法拆解 | 以單步驟模式直接執行 |

## 使用範例

- "幫我執行這個多步驟任務"
- "規劃一個工作流來完成這件事"
- "目前的工作流進度如何？"
