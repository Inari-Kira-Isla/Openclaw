# 狀態管理

管理長時間任務的狀態，支援進度記錄、中斷恢復與狀態查詢。

## 操作 / 工作流程

1. **狀態初始化** — 任務開始時建立狀態記錄，使用 `memory_search` 檢查是否有未完成任務
2. **進度更新** — 每個步驟完成後儲存檢查點（含完整上下文）
3. **狀態查詢** — 提供即時狀態查詢，回傳進度百分比與當前步驟
4. **中斷恢復** — 載入最近檢查點，從斷點繼續執行
5. **清理歸檔** — 成功完成 7 天後刪除，失敗 30 天後歸檔

### 狀態定義

| 狀態 | 說明 |
|------|------|
| pending | 等待開始 |
| running | 執行中 |
| paused | 已暫停 |
| completed | 已完成 |
| failed | 已失敗 |
| cancelled | 已取消 |

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| task_id | string | — | 任務唯一識別碼 |
| action | string | query | 操作：create / update / query / resume / cleanup |
| status | string | — | 目標狀態（用於 update） |
| context | object | {} | 步驟結果與變數（用於 update） |

## 輸出格式

```
📊 任務狀態：[task_id]
狀態：[running]
進度：[2/5] 40%
更新時間：[timestamp]
當前步驟：[步驟名稱]
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 找不到任務 ID | 回報任務不存在，建議建立新任務 |
| 檢查點損壞 | 回退到上一個有效檢查點 |
| 恢復失敗 | 記錄錯誤細節，建議從頭開始 |
| 狀態衝突 | 以最新時間戳為準 |

## 使用範例

- "這個任務進行到哪了？"
- "暫停目前的任務"
- "恢復上次中斷的任務"
- "清理已完成的舊任務"
