# 任務排程

安排任務的執行時間與優先順序，支援 cron 定時、延遲執行與條件觸發。

## 操作 / 工作流程

1. **接收排程請求** — 解析任務內容與時間需求
2. **優先級判定** — 依緊急-重要矩陣分級：
   | 等級 | 說明 | 處理方式 |
   |------|------|----------|
   | P0 | 立即執行，影響核心功能 | 馬上執行 |
   | P1 | 今天內完成 | 排入當日佇列 |
   | P2 | 本週完成 | 排程執行 |
   | P3 | 可延後 | 空閒時處理 |
3. **排程設定** — 使用 `cron` 工具設定定時任務，時區預設 `America/Los_Angeles`
4. **衝突檢測** — 避免同時段任務過多，分散執行時間
5. **執行通知** — 透過 `message` 工具在 Telegram 通知排程結果

### 排程類型

- **定時執行**：cron 表達式（如 `0 9 * * *` 每天早上 9 點）
- **延遲執行**：指定秒數後觸發
- **條件觸發**：特定任務完成後自動執行
- **循環執行**：固定間隔重複執行

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| task | string | — | 任務描述 |
| schedule_type | string | scheduled | scheduled / delayed / triggered / recurring |
| cron | string | — | cron 表達式（定時執行用） |
| delay | number | — | 延遲秒數（延遲執行用） |
| priority | string | P2 | 優先級：P0-P3 |
| timezone | string | America/Los_Angeles | 時區 |

## 輸出格式

```
⏱️ 排程已設定
任務：[任務描述]
類型：[定時執行]
時間：[cron 表達式 / 下次執行時間]
優先級：[P2]
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| cron 表達式無效 | 提示正確格式，建議修正 |
| 排程時間已過 | 詢問是否改為下一個週期 |
| 同時段任務過多 | 建議分散執行，列出衝突項 |
| 條件觸發的任務不存在 | 回報依賴任務缺失 |

## 使用範例

- "每天早上 9 點提醒我開會"
- "一小時後幫我發送訊息"
- "把這個任務排到明天處理"
- "這個任務完成後自動執行下一個"
