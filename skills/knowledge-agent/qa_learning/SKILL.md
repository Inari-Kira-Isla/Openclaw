# QA Learning — 問答學習

## 功能說明

自動從客戶互動中識別知識庫尚未覆蓋的新問題，生成候選答案，經人工審核後納入 FAQ 或知識庫，持續提升自動回覆覆蓋率。

## 學習流程

### 1. 新問題偵測
- 監控 faq_auto_reply 無法匹配的問題
- 收集 handoff_manager 轉人工的對話
- 標記重複出現 3 次以上的未知問題

### 2. 答案候選生成
- AI 根據上下文與知識庫生成候選答案
- 參考人工客服處理同類問題的回覆
- 關聯相似的已有 FAQ 條目

### 3. 人工審核
- 將候選 Q&A 提交給管理員審核
- 審核選項：通過 / 修改後通過 / 拒絕
- 通過後自動加入 FAQ 資料庫

### 4. 效果追蹤
- 追蹤新入庫 FAQ 的使用次數
- 監控自動回覆覆蓋率變化
- 定期報告學習成效

## 操作步驟

1. 定期掃描未匹配問題清單
2. 按出現頻率排序，優先處理高頻問題
3. 為每個問題生成 1-2 個候選答案
4. 提交審核請求至本地審核佇列（SQLite）
5. 審核通過後呼叫 faq_management 入庫
6. 一週後檢查新 FAQ 的使用效果

## 輸出格式

```
學習報告 — 本週新問題

待審核 Q&A（共 5 筆）：

1. [出現 8 次] Q: 你們接受什麼付款方式？
   候選 A: 我們接受信用卡、銀行轉帳與 LINE Pay。
   來源：Messenger 對話 #45, #52, #61...
   狀態：待審核

2. [出現 5 次] Q: 可以到店面看實品嗎？
   候選 A: 歡迎預約參觀！地址：...營業時間：...
   來源：Messenger 對話 #33, #48...
   狀態：待審核

本週自動回覆覆蓋率：72% → 預計入庫後可提升至 78%
```

## 工具指引

- **本地資料庫（SQLite）**：審核佇列、問題記錄資料庫
- **faq_management**：審核通過後入庫
- **conversation_logger**：未匹配對話來源
- **question_analytics**：問題頻率數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 候選答案品質低 | 標記為「需人工撰寫」，不自動入庫 |
| 問題屬於一次性 | 出現 < 3 次的問題不進入學習流程 |
| 審核佇列堆積 | 提醒管理員處理，按頻率排序優先審核 |
| 重複候選 | 自動合併相似問題，取出現頻率最高的問法 |

## 使用範例

- 「這週有哪些新問題需要審核？」
- 「自動回覆的覆蓋率現在多少？」
- 「把『付款方式』這個問題加進 FAQ」

## 防護規則

- 候選答案必須經人工審核才能入庫，不可自動發布
- 不可將客戶私人對話內容直接作為 FAQ 範例
- 學習資料需脫敏處理，移除個人身份資訊
- 候選答案中不可包含未經確認的業務承諾
- 拒絕的候選需記錄拒絕原因，避免重複生成
