# Knowledge Search — 知識搜尋

## 功能說明

提供知識庫的多模式搜尋功能，支援關鍵字精確搜尋與 AI 語義搜尋，協助快速找到所需資訊。

## 搜尋模式

### 1. 關鍵字搜尋
- **精確匹配**：完全符合關鍵字的結果
- **模糊匹配**：包含部分關鍵字或相近詞的結果

### 2. 語義搜尋
- 使用 AI 理解查詢意圖
- 即使措辭不同也能找到相關內容
- 依相關度排序

### 3. 進階篩選
- 依分類篩選（產品、服務、技術等）
- 依日期範圍篩選
- 依標籤過濾

## 操作步驟

1. 接收使用者的搜尋需求
2. 判斷最適合的搜尋模式（關鍵字 / 語義 / 進階）
3. 查詢本地資料庫（SQLite）
4. 依相關度排序結果
5. 輸出結果清單，每筆附摘要預覽
6. 若無結果，建議替代關鍵字或擴大搜尋範圍

## 輸出格式

```
搜尋結果（共 3 筆，關鍵字：「退換貨」）

1. 退換貨政策 [相關度：95%]
   摘要：七天鑑賞期內可無條件退換，需保持商品完整...
   分類：客服政策

2. 退款流程說明 [相關度：82%]
   摘要：退款將於收到退貨後 3-5 個工作天內處理...
   分類：帳務

3. 常見退換問題 FAQ [相關度：75%]
   摘要：Q: 超過七天還能退嗎？A: 視商品瑕疵情況...
   分類：FAQ
```

## 工具指引

- **本地資料庫（SQLite）**：知識庫資料庫查詢（主要工具）
- **faq_management**：FAQ 類資料來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 搜尋無結果 | 建議替代關鍵字，或提示知識庫可能尚未收錄 |
| 結果過多（> 10 筆） | 僅顯示前 5 筆最相關，提示可縮小搜尋範圍 |
| 本地資料庫查詢失敗 | 提示稍後重試 |
| 搜尋詞過於模糊 | 請使用者提供更具體的關鍵字或描述 |

## 使用範例

- 「搜尋一下退換貨相關的知識」
- 「知識庫裡有沒有關於 API 整合的文件？」
- 「找出所有標籤為『定價』的知識條目」

## 防護規則

- 搜尋結果僅顯示使用者有權限查看的內容
- 不可擅自修改知識庫內容，搜尋為唯讀操作
- 若搜尋內容涉及敏感資訊，需確認使用者權限
- 不可將搜尋結果直接作為對外回覆，需經過整理
