# Template Updater — 範本更新

## 功能說明

管理所有自動回覆範本的生命週期，從建立到測試到發布到效果追蹤，確保客服回覆品質一致且持續優化。

## 範本類型

| 類型 | 用途 | 範例 |
|------|------|------|
| 歡迎訊息 | 新用戶首次互動 | 「您好！歡迎來到...」 |
| FAQ 回覆 | 常見問題標準答案 | 價格、配送、退換貨 |
| 結束訊息 | 對話結束語 | 「感謝您的詢問...」 |
| 報告格式 | 數據報告模板 | 月報、週報格式 |
| 通知範本 | 系統通知訊息 | 提醒、警告、確認 |

## 操作步驟

### 新增範本
1. 確認範本類型與用途
2. 撰寫範本內容（支援變數插值 `{variable}`）
3. 設定觸發條件或關聯的關鍵字
4. 儲存為「草稿」狀態
5. 測試後發布

### 編輯範本
1. 查詢目標範本
2. 顯示現有內容
3. 修改指定部分
4. 儲存新版本（保留舊版本歷史）

### 範本生命週期
```
草稿 → 測試 → 發布 → 使用中 → 優化/停用
```

### 效果追蹤
- 每個範本記錄使用次數
- 追蹤客戶回覆率（互動是否繼續）
- 低效果範本標記為「需優化」

## 工具指引

- **Notion API**：範本資料庫 CRUD、版本歷史
- **faq_auto_reply**：FAQ 回覆範本的使用端
- **faq_management**：FAQ 內容來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 範本名稱重複 | 提示已存在，詢問是否更新或另建新名 |
| 變數未定義 | 列出範本中的變數，確認來源 |
| 發布前未測試 | 提醒先進行測試 |
| 舊範本仍有引用 | 停用前列出所有引用處，確認後再停用 |

## 使用範例

- 「新增一個價格詢問的回覆範本」
- 「更新歡迎訊息，改成更親切的語氣」
- 「哪些範本的使用率最低？」

## 防護規則

- 發布中的範本不可直接刪除，須先停用
- 編輯已發布範本會建立新版本，不覆蓋舊版
- 範本內容不可包含個人承諾或未經授權的促銷
- 範本語氣須符合品牌定位，不可使用非正式俚語
- 變更範本需記錄修改者與修改原因
