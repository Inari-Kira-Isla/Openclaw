# FAQ Management — FAQ 管理

## 功能說明

管理常見問題解答資料庫的完整生命週期，確保 FAQ 內容準確、分類清楚、持續更新，作為 faq_auto_reply 的核心資料來源。

## FAQ 結構

### 每筆 FAQ 包含：

| 欄位 | 必填 | 說明 |
|------|------|------|
| 問題 | 是 | 客戶常問的問題原文 |
| 答案 | 是 | 標準回覆內容 |
| 分類 | 是 | 所屬分類 |
| 關鍵字 | 是 | 觸發匹配的關鍵詞（可多個） |
| 相關連結 | 否 | 延伸閱讀或參考資料 |
| 標籤 | 否 | 自訂標籤 |
| 狀態 | 自動 | 草稿 / 已發布 / 已停用 |
| 使用次數 | 自動 | 被自動回覆使用的次數 |

### 預設分類

| 分類 | 涵蓋內容 |
|------|----------|
| 價格相關 | 定價、優惠、折扣、付款方式 |
| 產品服務 | 功能、規格、適用場景 |
| 物流配送 | 運費、配送時間、配送範圍 |
| 退換售後 | 退換貨政策、退款流程 |
| 技術支援 | 使用問題、故障排除 |
| 帳務問題 | 發票、收據、對帳 |

## 操作步驟

### 新增 FAQ
1. 確認問題、答案、分類、關鍵字
2. 檢查是否與現有 FAQ 重複或高度相似
3. 以「草稿」狀態寫入本地資料庫（SQLite）
4. 提示進行審核發布

### 編輯 FAQ
1. 查詢目標 FAQ
2. 顯示現有內容
3. 更新指定欄位
4. 記錄修改歷史

### 審核發布
1. 檢查答案的正確性與完整性
2. 確認關鍵字涵蓋常見問法
3. 將狀態改為「已發布」
4. 同步至 faq_auto_reply 使用

### 使用統計
1. 定期檢查各 FAQ 的使用次數
2. 標記低使用率條目（可能需優化關鍵字）
3. 標記高使用率條目（確認答案持續正確）

## 工具指引

- **本地資料庫（SQLite）**：FAQ 資料庫 CRUD（主要工具）
- **faq_auto_reply**：FAQ 的消費端
- **qa_learning**：新 FAQ 候選來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 重複 FAQ | 顯示相似條目，建議合併或區分差異 |
| 缺少必填欄位 | 逐一詢問：問題、答案、分類、關鍵字 |
| 答案內容過長 | 建議精簡為 3-5 句，複雜內容附連結 |
| 關鍵字覆蓋不足 | 建議增加同義詞與常見問法 |

## 使用範例

- 「新增一條 FAQ：客戶常問『你們週末有營業嗎？』」
- 「編輯退換貨的 FAQ，更新為新政策」
- 「哪些 FAQ 最少被使用？需要優化嗎？」

## 防護規則

- 已發布的 FAQ 修改後需重新審核
- 不可刪除有使用記錄的 FAQ，僅可停用
- 答案內容需經過事實確認，不可包含未驗證資訊
- FAQ 的定價資訊需與官方一致，修改需經授權
- 停用 FAQ 前需確認無其他技能依賴
