# Task Tracker — 任務追蹤

## 功能說明

追蹤所有討論的工作任務，從建立到完成的完整生命週期，確保每項工作都有明確的狀態與進度。

## 任務狀態流程

| 狀態 | 描述 | 下一步 |
|------|------|--------|
| 待處理 | 新建立的任務 | 開始處理 |
| 進行中 | 正在處理中 | 完成或受阻 |
| 待確認 | 等待用戶確認 | 確認後完成 |
| 已完成 | 任務完成 | 歸檔 |
| 已取消 | 任務取消 | 歸檔 |

## 優先級

| 等級 | 標記 | 說明 |
|------|------|------|
| P0 | 🔴 | 緊急且重要 |
| P1 | 🟠 | 重要但不緊急 |
| P2 | 🟡 | 一般任務 |
| P3 | 🟢 | 低優先級 |

## 操作步驟

### 新增任務
1. 確認任務標題與描述
2. 設定優先級（P0-P3）
3. 設定預計完成時間（可選）
4. 寫入 `memory/projects/tasks.md`

### 更新任務
1. 查詢指定任務
2. 更新狀態、進度百分比或備註
3. 記錄更新時間

### 查詢任務
1. 按狀態篩選（進行中、待處理等）
2. 按優先級篩選
3. 按日期範圍篩選

## 資料儲存

- **主要**：`memory/projects/tasks.md`
- **備份**：`memory/YYYY-MM-DD.md` 每日筆記
- **可選**：SQLite 任務資料庫

## 工具指引

- **檔案讀寫**：讀寫 tasks.md
- **記憶搜尋**：查詢歷史任務
- **提醒系統**：逾期任務提醒

## 使用範例

- 「記錄新任務：建立專案管理 agent，優先級 P1」
- 「更新任務 #5 狀態為已完成」
- 「顯示所有進行中的任務」
- 「這個月我完成了哪些任務？」

## 防護規則

- 任務狀態變更需使用者確認
- 不可刪除已完成任務，僅可標記或歸檔
- 涉及隱私的任務內容不對外分享
