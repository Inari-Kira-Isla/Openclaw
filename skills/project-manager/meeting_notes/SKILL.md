# Meeting Notes — 會議記錄

## 功能說明

記錄每次討論的重點，自動提取工作項目與決定事項，確保討論結論都被追蹤執行。

## 記錄結構

```
## 會議記錄 - [日期]
### 討論主題
[簡述討論內容]

### 決定事項
- [ ] 決定事項 1
- [ ] 決定事項 2

### 待辦事項
- [P1] 待辦事項 1 (負責人)
- [P2] 待辦事項 2 (負責人)

### 參考資料
- [相關連結或筆記]
```

## 操作步驟

### 建立記錄
1. 記錄討論日期與主題
2. 列出決定事項
3. 提取待辦事項（含優先級與負責人）
4. 儲存至 `memory/projects/meeting-notes/YYYY-MM-DD.md`

### 提取待辦
1. 掃描會議記錄
2. 識別待辦事項格式 `- [ ]` 或 `- [P1]`
3. 自動更新至 tasks.md

### 查詢歷史
1. 依日期或關鍵字搜尋
2. 列出相關決定事項
3. 顯示待辦狀態

## 資料儲存

- **會議記錄**：`memory/projects/meeting-notes/`
- **待辦事項**：整合至 `memory/projects/tasks.md`

## 工具指引

- **檔案系統**：建立/讀取 meeting-notes 目錄
- **記憶搜尋**：查詢歷史討論
- **任務追蹤**：連動 task_tracker

## 使用範例

- 「記錄今天的討論：關於升級系統，我們決定先修復 cron errors」
- 「從上次會議提取待辦事項」
- 「上週我們討論了什麼？」

## 防護規則

- 會議記錄需包含日期與參與者
- 決定事項必須明確
- 待辦事項需標註優先級與負責人
- 隱私內容不記錄或適當脫敏
