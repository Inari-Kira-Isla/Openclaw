# Progress Report — 進度報告

## 功能說明

從任務追蹤與會議記錄中提取數據，生成結構化的進度報告，幫助回顧與規劃。

## 報告類型

| 類型 | 頻率 | 內容 |
|------|------|------|
| 每日簡報 | 每日 | 今日任務、進度、障礙 |
| 週報 | 每週 | 本週完成、進行中、計劃 |
| 月報 | 每月 | 月度目標達成、學習、檢討 |
| 專案報告 | 按需 | 專案概況、里程碑、風險 |

## 生成步驟

### 1. 收集數據
- 查詢 tasks.md 任務狀態
- 查詢 meeting-notes 決定事項
- 計算完成率、逾期率

### 2. 分析趨勢
- 完成 vs 進行中比例
- 優先級分佈
- 瓶頸與障礙

### 3. 生成報告
- 選擇報告格式（簡潔/詳細）
- 包含統計數據與圖表（如適用）
- 提出建議

## 輸出格式

```markdown
## 📊 [報告類型] - [日期區間]

### 總覽
| 指標 | 數值 |
|------|------|
| 總任務數 | X |
| 已完成 | X |
| 進行中 | X |
| 完成率 | X% |

### 完成項目
- ✅ 項目 1
- ✅ 項目 2

### 進行中
- 🔄 項目 3 (80%)

### 障礙與建議
- [問題描述] → [建議]

### 下一步
- [ ] 任務 A
- [ ] 任務 B
```

## 工具指引

- **任務追蹤**：讀取 tasks.md
- **會議記錄**：讀取 meeting-notes
- **記憶搜尋**：查詢歷史數據

## 使用範例

- 「生成這週的週報」
- 「這個月的專案進度如何？」
- 「顯示所有進行中的任務進度」

## 防護規則

- 報告需基於實際數據，不可虛構
- 進度百分比需合理評估
- 建議需具體可行
- 敏感數據適當隱藏
