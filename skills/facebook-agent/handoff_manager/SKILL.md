# Handoff Manager

## 概述

管理從 AI Bot 轉接到人工客服的流程，確保對話順暢交接。

## 轉接觸發條件

### 自動轉接

| 條件 | 描述 |
|------|------|
| 關鍵詞 | 「我要跟真人講」、「轉人工」、「客服」 |
| 情緒檢測 | 偵測到負面情緒（抱怨、生氣） |
| 問題複雜度 | 超出 AI 處理能力 |
| 特定類型 | 訂單問題、財務問題、法律問題 |
| 多次失敗 | AI 回覆 3 次後仍無法解決 |

### 手動轉接

用戶明確要求轉接人工時。

## 轉接流程

```
1. 偵測轉接條件
2. 記錄對話歷史
3. 生成交接摘要
4. 通知值班人員
5. 轉移對話 thread
6. 標記對話狀態
7. 安排跟進
```

## 交接記錄格式

```markdown
## 📋 轉接交接單

### 用戶資訊
- 姓名：[姓名]
- FB ID：[ID）
- 訊息時間：[時間]

### 對話摘要
[AI 與用戶的對話重點]

### 轉接原因
[為什麼需要轉接人工]

### 待處理事項
- [ ] 聯繫用戶
- [ ] 解決問題
- [ ] 回覆結果

### 優先級
🔴 高 / 🟡 中 / 🟢 低
```

## 通知機制

### 轉接時通知

- Telegram 通知管理員
- Notion 建立待辦事項
- Email 通知（可選）

### 通知格式

```
🔔 新轉接請求

用戶：XXX
原因：XXX
對話：XXX
優先級：🔴 高
```

## 跟進處理

### 人工回覆後

1. 標記對話為「已處理」
2. 記錄解決方案
3. 同步到 Notion
4. 關閉待辦事項

### 未及時處理

- 30 分鐘未回覆 → 再次提醒
- 1 小時未回覆 → 升級通知

## 統計分析

追蹤轉接率：
- 總訊息數
- 轉接次數
- 轉接原因分布
- 平均處理時間

用於優化 AI 回覆能力。
