# Question Classifier — 問題分類

## 功能說明

對 Facebook Messenger 收到的客戶訊息進行即時分類，判定問題類型與優先級，供後續技能（faq_auto_reply、handoff_manager）決定處理方式。

## 分類維度

### 問題類型

| 類型 | 觸發關鍵詞 | 路由 |
|------|-----------|------|
| 價格 | 多少錢、報價、費用、價目表 | faq_auto_reply |
| 產品 | 功能、規格、這是什麼 | faq_auto_reply |
| 服務 | 有什麼服務、怎麼合作 | faq_auto_reply |
| 購買 | 怎麼買、下單、訂購 | faq_auto_reply |
| 投訴 | 抱怨、不滿意、出問題 | handoff_manager（轉人工） |
| 訂單 | 出貨、退款、退換貨 | handoff_manager（轉人工） |
| 其他 | 無法匹配以上類型 | AI 回覆 |

### 優先級

| 等級 | 條件 | 回應時限 |
|------|------|----------|
| P0 | 投訴、負面情緒強烈 | 立即轉人工 |
| P1 | 價格詢問、購買意向 | 5 分鐘內回覆 |
| P2 | 一般產品/服務詢問 | 30 分鐘內回覆 |
| P3 | 閒聊、無明確需求 | 有空再回覆 |

## 操作步驟

1. 接收客戶訊息原文
2. 進行關鍵詞比對，識別問題類型
3. 若關鍵詞無法判定，使用語義理解分析意圖
4. 評估情緒傾向（正面/中性/負面）
5. 綜合判定優先級
6. 輸出分類結果與路由建議

## 輸出格式

```
分類結果：
- 類型：價格
- 優先級：P1
- 情緒：中性
- 路由：faq_auto_reply
```

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 訊息過短（< 3 字） | 歸類為「其他」，P3，AI 回覆追問 |
| 多重意圖 | 以最高優先級的類型為主，附註其他意圖 |
| 非文字訊息（圖片/語音） | 標記為「需人工判斷」，轉 handoff_manager |
| 分類信心度低 | 標註「待確認」，預設以較高優先級處理 |

## 使用範例

- 「客戶傳來：『你們的方案多少錢？』」→ 類型：價格，P1
- 「客戶傳來：『我等了一週都沒收到！什麼爛服務』」→ 類型：投訴，P0
- 「客戶傳來：『你好～』」→ 類型：其他，P3

## 防護規則

- 不可直接回覆客戶，僅負責分類與路由
- 投訴類訊息一律標記 P0，不得降級
- 分類結果需附帶原始訊息，供下游技能參考
- 不可儲存或外洩客戶的個人身份資訊
