# Conversation Logger — 對話記錄

## 功能說明

將所有 Facebook Messenger 對話完整記錄至本地資料庫（SQLite: cs_customers.db），建立用戶檔案，並提供多維度檢索與匯出功能。

## 記錄內容

### 基本資訊

| 欄位 | 說明 |
|------|------|
| 用戶姓名 | Facebook 顯示名稱 |
| 用戶 ID | Facebook 用戶 ID |
| 粉絲專頁 | 接收訊息的粉絲專頁名稱 |
| 對話開始時間 | 第一則訊息時間 |
| 對話結束時間 | 最後一則訊息時間 |
| 狀態 | 進行中 / 已關閉 / 已轉接 |

### 對話內容
- 用戶訊息原文與 AI 回覆原文
- 對話輪次數量
- AI 生成的對話摘要

### 分析標籤
- 問題類型（來自 question_classifier）
- 情緒分析結果
- 處理結果（已解決 / 轉人工 / 未解決）

## 操作步驟

### 記錄對話
1. 接收新訊息事件（來自 facebook_messenger_handler）
2. 查詢本地資料庫（SQLite）是否已有該用戶的對話串
3. 若無，建立新用戶檔案與對話記錄
4. 若有，將新訊息追加至既有對話串
5. 更新對話狀態與分析標籤

### 檢索對話
1. 接收檢索條件（用戶、時間、標籤）
2. 查詢本地資料庫（SQLite）
3. 以摘要格式回覆結果

### 匯出資料
1. 確認匯出範圍與格式
2. 從本地資料庫（SQLite）拉取資料
3. 轉換為指定格式（Markdown / JSON / CSV）

## 工具指引

- **SQLite（cs_customers.db）**：對話資料庫的 CRUD 操作（主要工具，透過 cs_customer_db.py）
- **facebook_messenger_handler**：上游訊息來源
- **question_classifier**：分類標籤來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| SQLite 寫入失敗 | 將訊息暫存，排入重試佇列，不影響回覆流程 |
| 用戶 ID 無法解析 | 以匿名方式記錄，標記「待補充身份」 |
| 對話串過長 | 自動生成摘要，將舊訊息歸檔 |
| 匯出資料量過大 | 分批處理，每批最多 100 筆 |

## 使用範例

- 「查一下上週跟 XXX 的對話記錄」
- 「匯出本月所有投訴類對話」
- 「這個月總共有幾則 Messenger 對話？」

## 防護規則

- 自動遮蔽敏感資訊（信用卡號、身份證號等）
- 用戶要求刪除個人資料時須配合執行（GDPR 合規）
- 對話記錄僅限授權人員存取，不可公開
- 不可修改已記錄的原始對話內容
- 匯出操作需記錄操作者與用途
