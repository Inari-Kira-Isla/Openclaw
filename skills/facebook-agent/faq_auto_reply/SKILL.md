# FAQ Auto Reply — 常見問題自動回覆

## 功能說明

自動比對客戶問題與 FAQ 資料庫，找到最佳答案並回覆，減少人工客服負擔。無法匹配時觸發轉接。

## 匹配邏輯

### 第一層：關鍵詞匹配

| 關鍵詞 | 對應模板 |
|--------|----------|
| 價格/多少錢/報價/費用 | 價格資訊模板 |
| 怎麼買/購買/下單 | 購買流程模板 |
| 運費/配送/寄送/物流 | 配送資訊模板 |
| 退換/退款/退貨 | 退換貨模板 |
| 尺寸/規格/大小 | 產品規格模板 |
| 庫存/有沒有貨/缺貨 | 庫存查詢模板 |

### 第二層：AI 語義匹配
若關鍵詞無法匹配，使用 AI 理解用戶意圖，從 FAQ 資料庫中找出最相關的答案。

## 操作步驟

1. 接收經 question_classifier 分類為 FAQ 類的訊息
2. 進行關鍵詞匹配，嘗試找到對應模板
3. 若無匹配，啟用 AI 語義搜尋 FAQ 資料庫
4. 找到答案 → 套用模板格式回覆
5. 未找到答案 → 觸發 handoff_manager 轉人工
6. 記錄問答配對供學習機制使用

## 轉接條件

以下情況自動轉接人工（呼叫 handoff_manager）：
1. 問題不在 FAQ 範圍內且 AI 匹配信心度低
2. 客戶明確要求與真人對話
3. 同一客戶連續 3 次詢問未獲滿意回覆
4. 抱怨或投訴類訊息
5. 涉及訂單、退款等需要操作權限的問題

## 工具指引

- **Notion API**：讀取 FAQ 資料庫、記錄問答配對
- **question_classifier**：上游分類結果
- **handoff_manager**：無法回答時轉接

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| FAQ 資料庫為空 | 全部轉人工，並通知管理員建立 FAQ |
| 模板內容缺失 | 使用 AI 生成回覆，標記模板需修復 |
| 匹配到多個答案 | 回覆最相關的一個，附上「您也可能想問…」 |
| Notion 查詢失敗 | 以 AI 直接回覆，並記錄查詢失敗事件 |

## 使用範例

- 客戶問「你們怎麼收費？」→ 匹配「價格」→ 回覆價格模板
- 客戶問「東西多久會到？」→ 匹配「配送」→ 回覆物流模板
- 客戶問「你們公司在哪裡？」→ AI 語義匹配 → 回覆地址資訊

## 防護規則

- 回覆內容必須來自 FAQ 資料庫或經過驗證的模板，不可憑空編造
- 涉及價格時使用官方定價，不可自行打折或報價
- 不可在自動回覆中收集客戶的信用卡或敏感個資
- 回覆語氣需友善專業，符合品牌形象
- 配合 conversation_logger 記錄每次互動
