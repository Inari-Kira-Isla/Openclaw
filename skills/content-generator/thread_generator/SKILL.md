# 個人化 Thread 生成器

## 功能說明

在基礎 Thread 生成之上，融入用戶個人寫作風格，消除 AI 生成痕跡。基於 Wilson Huang 方法論，產出自然、有個人特色的內容。

## 工作流程

### 第一步：載入個人風格
- 檢查用戶的禁忌詞清單、常用詞、段落偏好、語氣特徵
- 若無設定，使用預設自然風格

### 第二步：主題分析
- 提煉核心論點、受眾痛點、獨特角度

### 第三步：結構規劃
- 熱點分析：數據 → 分析 → 預測
- 乾貨分享：問題 → 解決 → 案例
- 觀點表達：現象 → 觀點 → 論證
- 故事型：背景 → 衝突 → 轉折 → 結論

### 第四步：個人化內容生成
- 段落長短要有變化（有時一句、有時幾句）
- 避免 AI 常用句型（見禁忌清單）
- 融入用戶口頭禪和自嘲元素
- 結尾用問題引導互動

### 第五步：AI 味檢查
- 段落是否長度一致（不好）
- 是否使用固定結構「首先、其次、最後」（不好）
- 是否有對稱排比句（不好）
- 是否自然、像真人寫的（好）

## 禁忌詞清單（預設）

禁止使用：「首先」「其次」「最後」「值得注意的是」「不是 A 而是 B」「深入探討」「全面解析」「不僅...更...」

## 建議用詞（預設）

可使用：「說白了」「老實說」「其實」「根本上」

## 輸出格式

```
【標題】

【1/N】
內容（短，一兩句）

【2/N】
內容（稍長）

【3/N】
內容（突然轉折或自嘲）

【N/N】
[CTA - 問問題 / 呼籲互動]
```

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 用戶反饋「太 AI 味」 | 加入更多段落變化，減少工整感 |
| 用戶反饋「太正式」 | 加入口語化表達 |
| 用戶反饋「不像我」 | 請用戶提供寫作範本以學習風格 |
| 無個人風格設定 | 使用預設自然風格，首次生成後詢問調整方向 |

## 使用範例

- 「用我的風格寫一篇 AI 資本支出 Thread」
- 「生成 Thread，要像我平時講話那樣」
- 「爆款內容：加密貨幣監管，記得去 AI 味」

## 護欄

- 每篇 Thread 最多 20 篇
- 不捏造數據，需事實核查的標注來源
- 避免敏感話題
- 個人化風格僅調整語氣和用詞，不改變事實內容
- 不模仿其他公眾人物的寫作風格（除非用戶明確要求）
