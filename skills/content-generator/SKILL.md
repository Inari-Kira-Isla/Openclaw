# 爆款 Thread 生成器

## 功能說明

根據用戶提供的主題關鍵字，生成具有傳播潛力的 Twitter/X Thread 內容。涵蓋主題分析、結構規劃、內容撰寫及驗證檢查完整流程。

## 工作流程

### 第一步：主題分析
- 收到用戶輸入的主題關鍵字
- 提煉核心論點、受眾痛點、獨特角度
- 若主題不明確，主動追問釐清方向

### 第二步：選擇結構

| 類型 | 結構 | 適用場景 |
|------|------|----------|
| 熱點分析 | 數據 → 分析 → 預測 | 時事趨勢 |
| 乾貨分享 | 問題 → 解決 → 案例 | 教學知識 |
| 觀點表達 | 現象 → 觀點 → 論證 | 意見輸出 |
| 故事型 | 背景 → 衝突 → 轉折 → 結論 | 敘事內容 |

### 第三步：內容生成
- 標題鈎子：第一句話必須吸引注意力
- 數據支持：引用可靠來源增強可信度
- 視覺化：適當使用格式增強閱讀體驗
- CTA：最後一篇呼籲互動（留言、轉發、追蹤）

### 第四步：驗證檢查
- 數據來源是否可靠
- 觀點是否有獨特角度
- 語言是否生動有趣
- 格式是否易於閱讀
- 是否有互動呼籲

## 輸出格式

```
【標題】

【1/N】
內容...

【2/N】
內容...

【N/N】
[CTA - 呼籲互動]
```

## 工具指引

- 需要素材時呼叫 `information_gathering`
- 需要數據時呼叫 `analytics-agent`
- 需要事實核查時呼叫 `verification-agent`

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 主題過於模糊 | 請用戶提供更具體的關鍵字或角度 |
| 找不到可靠數據 | 標明「待查證」，不捏造數據 |
| 內容過長 | 自動拆分，每篇 Thread 控制在合理長度 |
| 涉及敏感話題 | 告知用戶該主題不適合，建議替代方向 |

## 使用範例

- 「幫我寫一篇關於 2026 AI 資本支出的 Thread」
- 「生成 Thread：為什麼遠端工作正在改變城市結構」
- 「爆款內容：加密貨幣監管趨勢分析」

## 護欄

- 每篇 Thread 最多 20 篇
- 不捏造數據或引用不存在的來源
- 避免政治、宗教等高度敏感話題
- 不生成仇恨言論或歧視性內容
- 需要事實核查的重要數據必須標注來源
