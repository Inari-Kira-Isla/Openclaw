# 歸檔管理

依據策略將過期資料歸檔壓縮，釋放儲存空間，並支援資料恢復。

## 操作 / 工作流程
1. 用 `browser` 掃描可歸檔資料：
   - 超過 30 天的日誌檔案
   - 已完成的任務紀錄
   - 過期的會話資料
   - 大型暫存檔案
2. 依時間策略分類：
   - 30 天以上 → 歸檔（壓縮保留）
   - 90 天以上 → 深度壓縮
   - 1 年以上 → 長期儲存或刪除候選
3. 執行歸檔：壓縮 → 移動到歸檔目錄 → 驗證完整性
4. 更新歸檔索引，記錄到 `memory_search`
5. 用 `message` 傳送歸檔摘要到 Telegram

## 參數
| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| action | string | scan | 動作：`scan`（僅掃描）/ `archive`（執行歸檔） |
| age | int | 30 | 歸檔門檻天數 |
| target | string | all | 目標：`all` / `logs` / `tasks` / `sessions` |

## 輸出格式
```
🗄️ 歸檔報告

掃描範圍：{scope}
可歸檔項目：{count} 個
預計釋放空間：{size}

| 類別 | 數量 | 大小 | 最舊日期 |
|------|------|------|----------|
| {category} | {count} | {size} | {oldest} |

{action == "archive" ? "✅ 已歸檔完成" : "⏳ 僅掃描，需確認後執行歸檔"}

歸檔目錄：{archive_path}
```

## 錯誤處理
| 錯誤 | 處理 |
|------|------|
| 磁碟空間不足以壓縮 | 先刪除可安全移除的暫存檔，再重試 |
| 壓縮驗證失敗 | 保留原始檔案，不刪除來源，回報錯誤 |
| 歸檔目錄不存在 | 自動建立目錄，回報路徑 |

## 使用範例
- "有哪些舊資料可以歸檔"
- "清理 90 天以上的日誌"
- "執行歸檔釋放空間"
