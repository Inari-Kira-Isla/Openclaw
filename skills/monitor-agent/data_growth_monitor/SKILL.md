# 資料成長監控

監控資料庫與檔案系統的容量使用、成長速率，預測空間耗盡時間並適時警報。

## 操作 / 工作流程
1. 用 `browser` 收集各儲存區的容量數據：
   - 磁碟使用量（總容量 / 已使用 / 可用）
   - SQLite 資料庫筆數與大小
   - 日誌檔案大小
2. 計算成長指標：每日成長量、每月成長率
3. 用 `memory_search` 取得歷史數據，計算成長趨勢
4. 預測空間耗盡日期
5. 若使用率超過閾值，用 `message` 傳送警報到 Telegram
6. 儲存本次數據到記憶，供趨勢分析使用

## 參數
| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| target | string | all | 監控目標：`all` / `disk` / `sqlite` / `logs` |
| threshold | int | 80 | 警報閾值（使用率百分比） |

## 輸出格式
```
📈 資料成長報告

| 儲存區 | 已用 | 總量 | 使用率 | 趨勢 |
|--------|------|------|--------|------|
| {name} | {used} | {total} | {pct}% | {trend} |

成長速率：{rate}/天
預計空間耗盡：{days_left} 天後 ({date})

⚠️ 警告：{warning_items}

建議：
- {recommendation}
```

## 錯誤處理
| 錯誤 | 處理 |
|------|------|
| 無法取得磁碟資訊 | 回報可取得的部分，標記無法檢查的儲存區 |
| 歷史數據不足 | 僅顯示當前狀態，標注「需累積數據才能預測」 |
| SQLite 無回應 | 跳過 SQLite 項目，回報其餘結果 |
