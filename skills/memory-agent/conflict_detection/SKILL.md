# 記憶衝突偵測

偵測新資訊與現有記憶之間的矛盾，標記衝突並提出解決建議。

## 操作 / 工作流程

1. **語義搜尋** — 用 `memory_search` 搜尋與新資訊相關的現有記憶（top 10）
2. **相似度比對** — LLM 逐一比對，判斷是否存在衝突：
   - 事實衝突（A 說紅色 vs B 說綠色）
   - 時間衝突（同一時段出現在不同地點）
   - 語義衝突（不喜歡 X vs 擁有 X）
3. **衝突分級** — 依嚴重程度分類：
   - `high`：直接矛盾，必須解決
   - `medium`：可能矛盾，需要確認
   - `low`：輕微不一致，可並存
4. **解決策略建議** — LLM 建議處理方式：
   - 覆蓋（信任新資訊）
   - 保留（兩者並存，標記待確認）
   - 詢問（透過 `message` 向用戶確認）
5. **執行解決** — 依策略更新記憶，記錄版本歷史

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| new_info | string | 必填 | 新輸入的資訊 |
| auto_resolve | boolean | false | 是否自動解決低衝突 |
| search_depth | number | 10 | 搜尋相關記憶的數量 |

## 輸出格式

```
⚠️ 衝突偵測報告
- 偵測到 {count} 個衝突

{foreach conflict}
  [{severity}] {description}
  舊記憶：{old_content}
  新資訊：{new_content}
  建議：{resolution}
{/foreach}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| memory_search 無回應 | 跳過衝突偵測，直接存入並標記「未檢查」 |
| 衝突數量過多（> 5） | 只顯示 high/medium 衝突，low 自動忽略 |
| 無法判斷衝突類型 | 標記為「待人工審查」，用 `message` 通知用戶 |

## 使用範例

- "Joe 的辦公室搬到信義區了"（可能與舊地址衝突）
- "更新：會議改到週五"（與現有排程衝突）
- "他其實不喝咖啡"（與偏好記憶衝突）
