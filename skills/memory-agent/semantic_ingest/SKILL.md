# 語義資料攝入

將原始資訊經過預處理、分塊、標記後存入記憶系統。

## 操作 / 工作流程

1. **預處理** — 清理原始文本，移除多餘空白與無關字元，標準化編碼
2. **結構化提取** — 用 LLM 提取關鍵欄位：
   - `content`：核心內容
   - `source`：來源（Telegram 對話 / 本地記憶 / 手動輸入）
   - `type`：事實 / 事件 / 人物 / 偏好 / 技能
   - `tags`：自動生成語義標籤
3. **分塊** — 依內容長度選擇策略：
   - 短文本（< 512 字元）：不分塊，整段存入
   - 長文本（> 512 字元）：語義分塊，保留上下文重疊
4. **存入記憶** — 呼叫 `memory_search` 的寫入介面，附帶 metadata
5. **驗證** — 用 `memory_search` 回查，確認資料已正確存入

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| content | string | 必填 | 要攝入的原始內容 |
| source | string | "manual" | 來源標識 |
| type | string | "fact" | 記憶類型：fact/event/person/preference/skill |
| tags | array | [] | 額外標籤 |
| priority | string | "normal" | 攝入優先級：low/normal/high |

## 輸出格式

```
📥 記憶已攝入
- ID：{memory_id}
- 類型：{type}
- 標籤：{tags}
- 分塊數：{chunk_count}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 內容為空 | 回報「無內容可攝入」，終止流程 |
| memory_search 寫入失敗 | 重試一次，仍失敗則回報錯誤並保留原始內容 |
| 重複內容偵測 | 提示已存在相似記憶，詢問是否仍要存入 |
| 內容過長（> 10000 字元） | 自動分塊處理，回報分塊數量 |

## 使用範例

- "記住這個：Joe 的生日是 3 月 15 號"
- "把這段會議紀錄存進記憶"
- "幫我存一下這個技術筆記"
