# 記憶衰減監控

定期掃描記憶庫，偵測衰減中的記憶並觸發刷新或歸檔。

## 操作 / 工作流程

1. **排程觸發** — 透過 `cron` 每日執行一次（建議凌晨 3:00 PST）
2. **掃描記憶** — 用 `memory_search` 取得所有記憶的 metadata，計算衰減狀態：
   - 健康（> 70% 強度）：無需處理
   - 衰減中（40-70%）：列入觀察清單
   - 臨界（< 40%）：需要刷新或歸檔
3. **刷新決策** — LLM 依重要性判斷：
   - 高重要性 + 臨界 → 自動刷新（更新時間戳與權重）
   - 低重要性 + 臨界 → 建議歸檔
   - 長期未被引用（> 90 天）→ 標記為候選歸檔
4. **執行動作** — 刷新或歸檔記憶
5. **產出報告** — 摘要通知

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| scan_scope | string | "all" | 掃描範圍：all / category / tag |
| decay_threshold | number | 0.4 | 臨界閾值 |
| auto_archive | boolean | true | 是否自動歸檔低重要性臨界記憶 |

## 輸出格式

```
📉 記憶健康報告
- 總記憶數：{total}
- 健康：{healthy} | 衰減中：{decayed} | 臨界：{critical}
- 今日刷新：{refreshed} 筆
- 今日歸檔：{archived} 筆
- 建議：{recommendations}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| cron 未觸發 | 下次觸發時補掃，記錄跳過次數 |
| 記憶庫無法存取 | 用 `message` 通知用戶，延後 1 小時重試 |
| 刷新數量過大（> 100） | 分批處理，每批 50 筆，間隔 5 分鐘 |

## 使用範例

- "檢查記憶健康狀態"
- "哪些記憶快要過期了"
- "刷新所有臨界記憶"
