# 記憶整理分類

對記憶庫進行分類、標籤整理、層次建構與歸檔管理。

## 操作 / 工作流程

1. **取得記憶清單** — 用 `memory_search` 取得待整理的記憶
2. **自動分類** — LLM 依內容分析，分配分類：
   - 時間維度：短期（< 1 週）/ 中期（1 週 - 1 年）/ 長期（> 1 年）
   - 類型維度：事實 / 事件 / 人物 / 偏好 / 技能
   - 來源維度：用戶提供 / 系統觀察 / 外部攝入
3. **標籤整理** — 統一標籤格式，合併同義標籤，建立層次結構
4. **聚合關聯** — 找出相似記憶進行聚合，建立記憶之間的連結
5. **歸檔處理** — 依規則歸檔：
   - 超過 1 年未存取 → 建議歸檔
   - 標記為過時 → 自動歸檔
   - 用戶要求刪除 → 完全移除
6. **更新 metadata** — 寫回分類與標籤到記憶系統

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| scope | string | "all" | 整理範圍：all / untagged / category:{name} |
| action | string | "organize" | 動作：organize / archive / merge |
| dry_run | boolean | false | 預覽模式，不實際修改 |

## 輸出格式

```
🗂️ 記憶整理報告
- 已分類：{categorized} 筆
- 標籤更新：{tagged} 筆
- 聚合：{merged} 組
- 歸檔：{archived} 筆
- 未處理：{skipped} 筆（原因：{reason}）
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 分類衝突（同一記憶符合多類別） | 選擇最佳匹配，次要分類存入標籤 |
| 記憶數量過大 | 分批處理，優先處理未分類記憶 |
| 標籤格式不一致 | 自動標準化，記錄變更日誌 |
| 歸檔後用戶需要恢復 | 支援從歸檔區還原 |

## 使用範例

- "整理一下記憶庫"
- "把沒有標籤的記憶分類一下"
- "歸檔超過半年沒用到的記憶"
