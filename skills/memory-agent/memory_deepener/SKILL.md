# 記憶深化

任務完成後對暫存思維和任務結果進行深化處理，提煉經驗規則並存入長期記憶。

## 操作 / 工作流程

1. **歸納提煉** — LLM 分析任務結果與暫存思維，提煉出經驗規則：
   - 輸入：任務內容 + active_thought_logger 的暫存思維
   - 輸出：一條精煉的經驗規則 + 標籤
2. **關聯連結** — 用 `memory_search` 搜尋相關舊記憶：
   - 找到相似記憶 → 建立連結關係
   - 找到相關專案 → 標記同屬關係
3. **衝突檢測** — 比對新舊記憶：
   - 衝突 → 標記「記憶更新」，保留新舊雙方
   - 矛盾 → 標記「資料矛盾」，提示需要驗證
   - 一致 → 正常儲存
4. **權重調整** — 計算重要性分數：
   - 驗證正確 +10 / 被引用 +5/次 / 決策相關 +15
5. **寫入記憶** — 透過 `memory_search` 寫入介面存入長期記憶

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| task_result | string | 必填 | 任務執行結果摘要 |
| temp_thoughts | array | [] | active_thought_logger 的暫存思維 |
| context | string | "" | 任務上下文 |

## 輸出格式

```
🔬 記憶深化完成
- 經驗規則：{summary}
- 標籤：{tags}
- 關聯記憶：{linked_count} 筆
- 衝突標記：{conflict_count} 個
- 重要性分數：{importance_score}/100
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 暫存思維為空 | 僅從 task_result 提煉，跳過思維整合 |
| memory_search 寫入失敗 | 重試一次，失敗則暫存到本地檔案待補寫 |
| 衝突過多無法自動解決 | 產出衝突報告，用 `message` 通知用戶審查 |
| 重要性分數異常（> 100） | 上限截斷為 100 |

## 使用範例

- "深化一下剛才的任務記憶"
- "把今天學到的經驗存起來"
- "整理暫存思維並歸檔"
