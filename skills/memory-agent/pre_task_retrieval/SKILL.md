# 任務前記憶提取

執行任務前進行意圖對齊檢索，同時使用語義搜尋與關鍵字搜尋，提供相關背景資訊。

## 操作 / 工作流程

1. **解析任務意圖** — LLM 從任務描述中提取：
   - 核心關鍵字（精確術語、專有名詞）
   - 語義主題（擴展相關概念）
2. **混合搜尋** — 同時執行兩種搜尋：
   - `memory_search` 語義搜尋（top 5，權重 0.7）
   - `memory_search` 關鍵字搜尋（top 5，權重 0.3）
3. **結果整合** — 合併去重，加權排序：
   ```
   final_score = semantic_score * 0.7 + keyword_score * 0.3
   ```
4. **相關性評估** — LLM 評估搜尋結果與當前任務的相關性：
   - 高度相關 → 直接附加到任務上下文
   - 中度相關 → 列為參考資訊
   - 不相關 → 丟棄，必要時擴大搜尋範圍
5. **輸出上下文包** — 將篩選後的記憶打包提供給任務執行者

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| task | string | 必填 | 任務描述 |
| context | string | "" | 額外背景資訊 |
| max_results | number | 5 | 最大回傳筆數 |
| min_relevance | number | 0.5 | 最低相關性閾值 |

## 輸出格式

```
🎯 任務前記憶提取
- 查詢：{expanded_query}
- 找到 {count} 筆相關記憶

{foreach memory}
  [{relevance_score}] {summary}
  來源：{source} | 日期：{date}
{/foreach}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 無相關記憶 | 回報「無相關經驗」，任務正常執行 |
| memory_search 超時 | 等待 5 秒後重試，仍失敗則跳過提取 |
| 結果全部低相關性 | 擴大搜尋範圍（降低閾值到 0.3），再無結果則放棄 |
| 任務描述過短 | 請求補充描述，或從上下文推斷意圖 |

## 使用範例

- "準備一下關於 BNI 會議的背景資料"
- "上次處理 Facebook API 的經驗是什麼"
- "找一下之前豐洲市場報價的相關記憶"
