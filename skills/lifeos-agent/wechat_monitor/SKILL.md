# WeChat Monitor

## 功能說明

透過 browser tool 定時截取 WeChat 網頁版畫面，偵測對話列表中的新訊息，並透過 Telegram 即時通知用戶。此功能僅供監控，不自動回覆訊息。

## 前置條件

- WeChat 網頁版（web.wechat.com）已開啟且保持登入
- browser tool 可用
- Telegram 通知已設定

## 執行步驟

### 1. 啟動監控

- 確認 WeChat 網頁版已開啟並登入
- 設定監控參數（間隔、安靜時段）
- 執行首次截圖作為基準線

### 2. 定時截圖

- 依設定間隔截取 WeChat 網頁版畫面
- 記錄對話列表目前狀態（聯絡人排列、未讀標記）

### 3. 新訊息偵測

- 比對前後兩次截圖，偵測變化
- 識別有新訊息的對話（未讀標記、排序變動）
- 辨識發訊人名稱與訊息預覽文字

### 4. 推送通知

偵測到新訊息時，透過 Telegram 推送：

```
WeChat 新訊息

來自：[聯絡人名稱]
內容：[訊息預覽摘要]
時間：[偵測時間]
```

## 監控參數

| 參數 | 預設值 | 說明 |
|------|--------|------|
| 截圖間隔 | 60 秒 | 每次檢查的間隔 |
| 新訊息通知 | 開啟 | 偵測到新訊息時推送 |
| 安靜時段 | 22:00-07:00 PST | 此時段內不推送通知 |

## 工具指令

- **browser tool**：截取 WeChat 網頁版畫面
- **Telegram API**：推送新訊息通知
- **cron**：設定定時截圖排程

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| WeChat 網頁版已登出 | 立即通知用戶「WeChat 已登出，請重新掃碼登入」，暫停監控 |
| 截圖失敗 | 重試 1 次，仍失敗則通知用戶並暫停 |
| 網頁無回應 | 等待 30 秒後重試，連續 3 次失敗則通知用戶 |
| browser tool 不可用 | 通知用戶「監控工具不可用」，停止排程 |

## 使用範例

- 「開始監控 WeChat」
- 「暫停 WeChat 監控」
- 「WeChat 監控改成每 30 秒檢查一次」

## 安全邊界

- 僅監控，絕不自動回覆或發送訊息
- 不儲存完整截圖，僅記錄偵測到的訊息摘要
- 安靜時段內不推送通知，但仍可記錄新訊息待早上彙整
- 不擷取或轉發訊息中的圖片、檔案等附件內容
- 用戶說「停止監控」時必須立即停止所有排程
