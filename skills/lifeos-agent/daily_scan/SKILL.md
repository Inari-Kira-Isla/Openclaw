# Daily Scan

## 功能說明

執行每日例行資訊掃描，彙整郵件、行程、待辦、天氣、交通等資訊，產出每日簡報推送給用戶。可由 cron 排程自動觸發或用戶手動請求。

## 執行步驟

### 1. 郵件掃描

- 檢查未讀郵件數量
- 篩選重要郵件（含會議邀請、緊急標記）
- 摘要前 3 封重要郵件的主旨與寄件者

### 2. 行程彙整

- 查詢本地資料庫（SQLite）取得今日行程
- 列出時間、標題、地點
- 標記即將開始的行程（1 小時內）

### 3. 待辦檢查

- 查詢本地待辦資料庫（SQLite）
- 列出今日到期的待辦事項
- 標記逾期未完成的任務

### 4. 天氣與交通

- 取得當地天氣概況（溫度、天況）
- 取得通勤路線交通狀況（如有設定）

### 5. 產出簡報

```
每日簡報 - [日期]

郵件：X 封未讀（Y 封重要）
行程：X 個
待辦：X 項（Y 項逾期）

天氣：[天況]，[溫度]
交通：[狀況]

重要事項：
- [事項1]
- [事項2]
```

## 工具指令

- **SQLite**：查詢日程與待辦資料庫
- **cron**：每日自動觸發（建議 07:00 PST）
- **Telegram API**：推送簡報給用戶

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 資料庫查詢失敗 | 跳過該區塊，簡報中標示「行程/待辦暫時無法取得」 |
| 天氣 API 失敗 | 跳過天氣區塊，不阻擋簡報推送 |
| 無任何行程或待辦 | 正常產出簡報，顯示「今日無行程 / 無待辦」 |
| 排程未觸發 | 用戶可手動說「給我今天的簡報」觸發 |

## 使用範例

- 「今天有什麼事？」
- 「給我每日簡報」
- （cron 自動觸發，每天早上 7 點推送）

## 安全邊界

- 簡報為唯讀彙整，不得在掃描過程中修改任何資料
- 不主動標記郵件為已讀
- 天氣與交通資訊僅供參考，不做決策建議
- 若掃描過程有部分失敗，仍需推送可用部分的簡報
- 安靜時段（22:00-07:00 PST）不推送，除非用戶手動請求
