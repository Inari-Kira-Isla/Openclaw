# Information Gathering

## 功能說明

根據用戶需求進行資訊收集與研究，涵蓋產業趨勢、競爭分析、技術資訊等主題。搜尋後整理為結構化摘要，附上來源供用戶參考。

## 執行步驟

### 1. 確認研究範圍

- 從用戶訊息提取研究主題與關鍵字
- 確認研究深度（快速摘要 / 深入報告）
- 確認是否有特定來源偏好或排除條件

### 2. 資料搜尋

- 使用搜尋工具查詢相關資料
- 優先選擇權威來源（官方文件、學術機構、知名媒體）
- 收集至少 3 個不同來源的資訊

### 3. 來源驗證

- 檢查資訊發布日期，標記過時資訊
- 交叉比對多個來源，排除矛盾或不可靠內容
- 標注資訊可信度（高/中/低）

### 4. 整理輸出

```
[主題] 研究摘要

重點發現：
1. [發現1]
2. [發現2]
3. [發現3]

來源：
- [來源1]（可信度：高）
- [來源2]（可信度：中）

建議閱讀：
- [連結1] - [簡述]
- [連結2] - [簡述]
```

### 5. 儲存記錄（選填）

- 若用戶要求，將研究結果存入本地資料庫（SQLite）備查

## 工具指令

- **web_search**：網路搜尋取得最新資訊
- **web_fetch**：擷取特定網頁內容
- **SQLite**：儲存研究結果（選填）

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 主題太模糊 | 追問「能否更具體描述你想了解的方向？」 |
| 搜尋無結果 | 建議替代關鍵字或調整搜尋範圍 |
| 來源全部過時 | 告知「目前找到的資訊較舊，建議參考時注意時效」 |
| 資訊互相矛盾 | 列出不同觀點並標注來源，讓用戶自行判斷 |
| 搜尋工具不可用 | 改用既有知識回答，並標明「此為模型知識，非即時搜尋」 |

## 使用範例

- 「幫我查一下 AI Agent 的最新趨勢」
- 「研究一下洛杉磯最近的房價走勢」
- 「找一下 n8n 和 Make 的比較」

## 安全邊界

- 不捏造來源或引用不存在的資料
- 明確區分「事實」與「觀點」
- 不提供醫療、法律、財務等專業建議，僅做資訊整理
- 若資訊不足以得出結論，如實告知而非強行推論
- 不存取用戶未授權的私人資料來源
