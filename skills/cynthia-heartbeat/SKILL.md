# Cynthia Heartbeat — 每小時學習與系統優化

## 功能說明

每小時觸發一次，分析最近對話、提取知識、更新系統。

## 觸發條件

- Heartbeat：每小時 (`0 * * * *`)

## 工作流程

### 步驟 1：對話摘要
```
1. 讀取最近 1 小時的所有對話記錄
2. 提取：發言者、關鍵字、操作請求、決策
3. 濃縮成結構化筆記
```

### 步驟 2：知識提取
```
- 辨識新知識點（技術、操作、偏好）
- 檢查是否與現有知識重複
- 標註需要入庫的新內容
```

### 步驟 3：系統優化建議
```
- 識別重複問題 → 建議更新 FAQ
- 發現系統不足 → 產生優化提案
- 記錄決策過程 → 形成案例庫
```

### 步驟 4：儲存與輸出
```
- 寫入 ~/.openclaw/workspace/memory/learning/YYYY-MM-DD-HH.md
- 產出優化建議報告（如果有）
```

## 輸出格式

```
# 學習筆記 - {YYYY-MM-DD HH:00}

## 對話摘要
| 發言者 | 內容摘要 | 操作 |
|--------|----------|------|
| Joe | 請求優化系統 | 記錄 |

## 新知識點
- 知識點 1

## 優化建議
- 建議 1

## 待處理
- [ ] 待處理項目
```

## 工具指引

- **sessions_list**: 查詢最近對話
- **sessions_history**: 讀取對話詳情
- **memory_search**: 搜尋現有知識
- **write**: 寫入學習筆記

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 無新對話 | 輸出「無新對話，進入休眠」 |
| 寫入失敗 | 回報錯誤，保留記憶體緩衝 |
| 知識庫連線失敗 | 跳過搜尋，繼續摘要 |

## 確認規則

- 系統優化建議需要用戶確認後才能執行
- 危險操作（T2+）必須明確獲得許可
