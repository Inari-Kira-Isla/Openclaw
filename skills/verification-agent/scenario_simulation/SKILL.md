# 情境模擬

設計並執行模擬測試情境，驗證技能與工作流在各種條件下的正確性。

## 操作 / 工作流程

1. **情境設計** — 依測試目標設計情境，涵蓋四種類型：
   | 類型 | 說明 | 範例 |
   |------|------|------|
   | Happy Path | 正常流程 | 查詢天氣 → 正確顯示 |
   | Edge Case | 邊界情況 | 空輸入 → 適當提示 |
   | Error Case | 錯誤情況 | 網路斷線 → 錯誤訊息 |
   | Complex | 複雜情境 | 多步驟任務 → 正確串接 |
2. **測試案例建立** — 定義前提條件、操作步驟、預期結果
3. **執行模擬** — 逐一或批量執行測試案例，記錄實際結果
4. **結果比對** — 比對預期與實際結果，標記通過/失敗
5. **報告產出** — 彙整測試結果，失敗項交由 `failure_analysis` 分析

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| target | string | — | 測試目標（技能名稱或工作流 ID） |
| scenarios | array | [] | 自訂測試情境列表 |
| mode | string | single | 執行模式：single / batch / chain |
| auto_generate | boolean | true | 是否自動生成測試情境 |

## 輸出格式

```
🎭 情境模擬報告
目標：[測試目標]
總數：[10] 個情境
通過：[8] ✅
失敗：[2] ❌

失敗項目：
1. [情境名稱] — [失敗原因]
2. [情境名稱] — [失敗原因]
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 測試目標不存在 | 回報目標缺失，列出可用目標 |
| 情境執行逾時 | 標記該情境為逾時失敗，繼續下一個 |
| 預期結果格式錯誤 | 提示修正預期結果格式 |
| 批量測試中途崩潰 | 儲存已完成結果，回報崩潰點 |

## 使用範例

- "測試這個技能在各種情境下是否正常"
- "模擬用戶輸入錯誤的情況"
- "跑一輪完整的情境測試"
