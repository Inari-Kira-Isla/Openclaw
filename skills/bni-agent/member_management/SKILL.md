# Member Management — 會員管理

## 功能說明

管理 BNI 分會所有會員的基本資料、分類標籤與會籍狀態，作為其他 BNI 技能的基礎資料來源。

## 會員資料結構

| 欄位 | 必填 | 說明 |
|------|------|------|
| 姓名 | 是 | 會員全名 |
| 公司 | 是 | 所屬公司名稱 |
| 職位 | 是 | 職稱 |
| 產業類別 | 是 | 所屬產業分類 |
| 電話 | 否 | 聯絡電話 |
| Email | 否 | 電子郵件 |
| 會籍到期日 | 是 | 會籍期限 |
| 標籤 | 否 | 自訂分類標籤 |
| 狀態 | 是 | 活躍 / 暫停 / 已退會 |

## 操作步驟

### 新增會員
1. 收集必填欄位：姓名、公司、職位、產業類別、會籍到期日
2. 確認無重複會員（姓名 + 公司比對）
3. 寫入 Notion 會員資料庫
4. 回覆確認訊息

### 查詢會員
1. 接收查詢條件（姓名、產業、標籤等）
2. 從 Notion 查詢符合結果
3. 以表格格式回覆

### 編輯會員
1. 先查詢確認目標會員
2. 顯示目前資料供使用者確認
3. 更新指定欄位
4. 回覆更新後的完整資料

### 會籍提醒
1. 每月檢查到期日在 30 天內的會員
2. 發送提醒通知給管理者

## 工具指引

- **Notion API**：會員資料庫的 CRUD 操作
- **提醒系統**：會籍到期通知

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 重複會員 | 提示已有同名會員，顯示既有資料供比對 |
| 缺少必填欄位 | 逐一詢問缺少的欄位 |
| 查無結果 | 建議放寬搜尋條件或確認拼寫 |
| Notion 連線失敗 | 提示稍後重試 |

## 使用範例

- 「新增會員：張三，ABC 科技，技術總監，資訊業」
- 「查一下保險業的會員有哪些」
- 「更新王小明的電話為 0912-345-678」

## 防護規則

- 不可批次刪除會員資料，僅可將狀態改為「已退會」
- 修改會員資料需使用者明確確認後才執行
- 會員個人聯絡資訊（電話、Email）不可對外公開
- 不可在未經授權下匯出完整會員清單
