# Analytics Report — BNI 數據報告

## 功能說明

彙整 BNI 分會各項業務數據，生成結構化報告，協助會員與幹部掌握績效表現與發展趨勢。

## 報告類型

| 類型 | 內容 | 建議週期 |
|------|------|----------|
| 轉介紹報告 | 收到/給出統計、成交率、產業分布 | 每週 |
| 會員報告 | 出席率、活躍度、參與度排名 | 每月 |
| 營收報告 | 業績統計、目標達成率、成長趨勢 | 每月/每季 |

## 操作步驟

1. 確認報告類型與時間範圍（預設：當月）
2. 從本地資料庫（SQLite: bni.db）拉取原始數據
3. 計算統計指標（成交率、成長率、排名等）
4. 組合為結構化報告
5. 以使用者偏好的格式輸出

## 輸出格式

### 文字摘要
```
BNI 月報 — 2024 年 1 月

轉介紹：收到 15 筆 / 給出 12 筆 / 成交 8 筆（成交率 53%）
會員出席率：92%（32/35 人）
本月營收貢獻：$320,000（目標達成率 80%）
較上月成長：+12%
```

### 表格格式
依需求提供排名表、分布表或趨勢對比表。

## 工具指引

- **SQLite（bni.db）**：拉取轉介紹記錄、會員出席、營收數據（透過 bni_db.py）
- **referral_tracker**：轉介紹成交數據
- **member_management**：會員活躍度數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 數據不足 | 說明可用數據範圍，以現有數據產出部分報告 |
| 時間範圍無資料 | 提示該期間無記錄，建議調整範圍 |
| 資料庫查詢失敗 | 提示稍後重試 |
| 指標計算異常 | 標註異常值並說明可能原因 |

## 使用範例

- 「幫我拉這個月的轉介紹報告」
- 「上季度的營收目標達成率多少？」
- 「哪位會員這個月給出最多轉介紹？」

## 防護規則

- 報告中不得包含會員私人聯絡資訊
- 營收數據僅供內部參考，不可對外發布
- 排名報告需中立呈現，不加入主觀評論
- 數據不確定時需標註「估算值」或「資料不完整」
