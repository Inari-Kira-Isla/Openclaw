# Referral Tracker — 轉介紹追蹤

## 功能說明

追蹤 BNI 會員之間的轉介紹從接收到成交的完整生命週期，確保每筆轉介紹都有明確的狀態與跟進。

## 轉介紹狀態流程

| 狀態 | 描述 | 下一步 |
|------|------|--------|
| 剛收到 | 新收到的轉介紹 | 72 小時內聯繫 |
| 聯繫中 | 已聯繫對方，待進一步溝通 | 記錄溝通結果 |
| 報價中 | 已提交報價，等待回覆 | 一週內跟進 |
| 成交 | 成功成交，記錄金額 | 回報結果給推薦人 |
| 未成交 | 結束但未成交 | 記錄原因、感謝推薦人 |

## 操作步驟

### 新增轉介紹
1. 確認推薦人（給出方）與受益人（接收方）
2. 記錄產品/服務類型與預估金額
3. 設定初始狀態為「剛收到」
4. 寫入 Notion 轉介紹資料庫

### 更新狀態
1. 查詢指定轉介紹記錄
2. 更新狀態欄位與備註
3. 若成交，記錄實際金額與成交日期
4. 若未成交，記錄失敗原因

### 統計查詢
1. 依會員、時間範圍或狀態篩選
2. 計算成交率、平均金額、轉介紹淨值
3. 以表格或摘要格式回覆

## 工具指引

- **Notion API**：讀寫轉介紹資料庫（主要資料源）
- **提醒系統**：逾期未跟進時發送提醒

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| Notion 寫入失敗 | 提示使用者稍後重試，暫存資料於對話中 |
| 找不到會員 | 確認姓名拼寫，建議先用 member_management 新增 |
| 重複記錄 | 提示已有相同轉介紹，詢問是否更新 |
| 缺少必填欄位 | 逐一詢問：推薦人、受益人、產品類型 |

## 使用範例

- 「記錄一筆轉介紹：王大明推薦李小華，網站設計，預估 5 萬」
- 「更新轉介紹 #12 狀態為成交，金額 48000」
- 「這個月我收到幾筆轉介紹？成交率多少？」

## 防護規則

- 不可刪除已成交的轉介紹記錄，僅可標記或封存
- 金額與狀態變更需使用者確認後才寫入
- 不可代替使用者虛報轉介紹數據
- 涉及金額資訊時不對外分享，僅限當事人查看
