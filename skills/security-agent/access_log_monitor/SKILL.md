# 存取日誌監控

持續監控系統存取日誌，偵測異常登入、異常流量與可疑行為，即時發出警報。

## 操作 / 工作流程
1. 用 `browser` 讀取系統存取日誌（`~/.openclaw/logs/`）
2. 解析每筆存取紀錄：時間、來源 IP、操作類型、結果（成功/失敗）
3. 套用異常偵測規則：
   - 5 分鐘內失敗登入 >= 5 次
   - 凌晨 0-5 點的存取
   - 單一來源高頻請求 >= 100 次/分鐘
4. 彙整統計（總次數、成功率、失敗率）
5. 若偵測到異常，立即用 `message` 傳送警報到 Telegram
6. 將監控結果寫入 `memory_search` 供後續查詢

## 參數
| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| period | string | today | 查詢範圍：`today` / `week` / `custom` |
| alert_only | bool | false | 僅顯示異常項目 |

## 輸出格式
```
📝 存取日誌報告 — {date}

總存取次數：{total}
✅ 成功：{success} ({success_pct}%)
❌ 失敗：{fail} ({fail_pct}%)

⚠️ 異常偵測：
- {anomaly_type}: {description} (來源: {source})

近期趨勢：{trend_summary}
```

## 錯誤處理
| 錯誤 | 處理 |
|------|------|
| 日誌檔案不存在 | 通知管理員，確認日誌路徑設定 |
| 日誌格式異常 | 跳過無法解析的行，記錄數量回報 |
| 日誌檔案過大 | 僅分析最近 24 小時，建議執行歸檔 |

## 使用範例
- "今天有什麼異常存取嗎"
- "查看這週的登入紀錄"
- "誰在凌晨存取了系統"
