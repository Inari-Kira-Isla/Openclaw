# Prompt Refinement

## 診斷維度

### 1. 模糊性
- 指令是否清晰
- 是否有歧義
- 上下文是否足夠

### 2. 完整性
- 是否覆蓋所有情況
- 邊界條件是否處理
- 是否有遺漏

### 3. 一致性
- 風格是否統一
- 指令是否矛盾
- 範例是否一致

### 4. 有效性
- 是否有幫助
- 是否過長
- 是否有廢話

## 優化策略

### 結構化
```
## 角色
[定義 AI 角色]

## 任務
[明確說明任務]

## 輸出格式
[指定輸出結構]

## 範例
[提供範例]
```

### 清晰化
- 使用明確的動詞
- 避免模糊詞彙
- 提供具體例子

### 簡潔化
- 移除冗餘資訊
- 合併相似指令
- 使用清單格式

## 測試驗證

### A/B 測試
```json
{
  "variant_a": "原始 prompt",
  "variant_b": "優化後 prompt",
  "test_duration": "1 週",
  "metrics": ["準確率", "滿意度", "執行時間"]
}
```

### 評估指標
- 任務完成率
- 輸出品質評分
- 用戶滿意度

## 版本管理
- 每次優化記錄版本
- 保留歷史版本
- 標記棄用版本
