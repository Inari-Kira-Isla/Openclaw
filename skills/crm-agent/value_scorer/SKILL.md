# Value Scorer — 客戶價值評分

## 功能說明

根據多維度指標為每位客戶計算價值分數，分級為 A/B/C/D 等級，作為跟進優先級與資源分配的依據。

## 評分維度

| 維度 | 權重 | 評估指標 |
|------|------|----------|
| 購買力 | 30% | 營收規模、歷史交易金額、預算範圍 |
| 購買意願 | 25% | 詢問頻率、回覆速度、主動聯繫次數 |
| 戰略價值 | 20% | 產業相關性、品牌曝光效益、介紹潛力 |
| 互動頻率 | 15% | 近 30 天互動次數、回應率 |
| 成長潛力 | 10% | 公司成長趨勢、擴展計畫 |

## 等級定義

| 等級 | 分數範圍 | 描述 | 建議策略 |
|------|----------|------|----------|
| A | 80-100 | 高價值客戶 | 優先跟進，專人維護，提供客製方案 |
| B | 60-79 | 中價值客戶 | 定期聯繫，維持關係，適時推薦 |
| C | 40-59 | 低價值客戶 | 自動化互動為主，偶爾主動聯繫 |
| D | 0-39 | 觀察期客戶 | 低優先，持續觀察，等待時機 |

## 操作步驟

1. 收集客戶基本資料與歷史互動數據
2. 依各維度評分（每維度 0-100 分）
3. 加權計算總分
4. 對照等級表分級
5. 輸出評分結果與策略建議
6. 寫入 Notion 客戶資料庫

### 定期重評
- 每月自動重新計算所有客戶分數
- 等級變動時通知業務負責人

## 輸出格式

```
客戶評分結果：
客戶：[名稱]
總分：78 / 100（等級：B）
- 購買力：85
- 購買意願：70
- 戰略價值：80
- 互動頻率：65
- 成長潛力：75
建議：定期聯繫，適時推薦進階方案
```

## 工具指引

- **Notion API**：讀取客戶資料與互動歷史、寫入評分結果
- **interaction_tracker**：取得互動頻率數據
- **customer_analytics**：補充行為分析數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 資料不足（新客戶） | 僅評估已有維度，其餘以中位數填入，標註「資料不足」 |
| 互動歷史為空 | 互動頻率設為 0，建議先建立初次接觸 |
| 計算結果異常 | 檢查原始數據，標註異常項，人工覆核 |
| Notion 查詢失敗 | 提示稍後重試 |

## 使用範例

- 「幫我評估一下 ABC 公司的客戶價值」
- 「哪些客戶是 A 級？列出來」
- 「這個月有哪些客戶等級發生變動？」

## 防護規則

- 評分結果為內部參考，不可告知客戶本人
- 不可僅依單一維度決定等級，須綜合評估
- 人工可覆寫自動評分，但需記錄覆寫原因
- D 級客戶不可自動刪除，僅降低跟進頻率
