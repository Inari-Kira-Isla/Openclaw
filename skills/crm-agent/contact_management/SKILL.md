# Contact Management — 聯絡人管理

## 功能說明

管理 CRM 系統中所有客戶聯絡人的基本資料與業務屬性，作為 CRM 其他技能的核心資料來源。

## 聯絡人資料結構

### 基本資料

| 欄位 | 必填 | 說明 |
|------|------|------|
| 姓名 | 是 | 聯絡人全名 |
| 公司 | 是 | 所屬公司 |
| 職位 | 否 | 職稱 |
| 電話 | 否 | 聯絡電話 |
| Email | 否 | 電子郵件 |

### 業務屬性

| 欄位 | 必填 | 說明 |
|------|------|------|
| 客戶等級 | 否 | A/B/C/D（由 value_scorer 評定） |
| 產業別 | 是 | 產業分類 |
| 來源 | 是 | BNI / Facebook / 轉介紹 / 自來 |
| 業務負責人 | 否 | 負責跟進的業務 |
| 標籤 | 否 | 自訂分類標籤（可多個） |
| 建立日期 | 自動 | 系統自動記錄 |

## 操作步驟

### 新增聯絡人
1. 收集必填欄位：姓名、公司、產業、來源
2. 確認無重複（姓名 + 公司比對）
3. 寫入 Notion 聯絡人資料庫
4. 回覆新增成功確認

### 查詢聯絡人
1. 接收查詢條件（姓名、公司、產業、標籤、來源）
2. 從 Notion 查詢符合結果
3. 以清單或表格格式回覆

### 編輯聯絡人
1. 先查詢確認目標聯絡人
2. 顯示現有資料
3. 更新指定欄位
4. 回覆更新結果

### 批次操作
1. 支援依條件批次加標籤
2. 支援 CSV 格式匯入/匯出

## 工具指引

- **Notion API**：聯絡人資料庫 CRUD（主要工具）
- **value_scorer**：客戶等級數據來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 重複聯絡人 | 顯示既有資料，詢問是否更新或新增（不同人同名） |
| 缺少必填欄位 | 逐一詢問：姓名、公司、產業、來源 |
| 查無結果 | 建議放寬條件或確認拼寫 |
| 匯入格式錯誤 | 說明正確 CSV 格式，提供範例 |
| Notion 連線失敗 | 提示稍後重試 |

## 使用範例

- 「新增客戶：張三，ABC 科技，資訊業，來源 BNI」
- 「查一下我的 A 級客戶有哪些」
- 「把所有 Facebook 來源的客戶加上 #social 標籤」

## 防護規則

- 不可批次刪除聯絡人，僅可標記為「停用」
- 編輯操作需使用者確認後執行
- 聯絡人私人資訊（電話、Email）不可對外公開
- 匯出完整聯絡人清單需經授權確認
- 不可在未經同意下將聯絡人資料分享給第三方
