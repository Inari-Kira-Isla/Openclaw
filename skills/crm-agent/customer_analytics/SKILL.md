# Customer Analytics — 客戶分析

## 功能說明

從 SQLite CRM 資料庫中彙整客戶數據，進行多維度分析，產出可執行的商業洞察，協助優化客戶經營策略。

## 分析維度

### 1. 客戶分布分析
- 產業分布：各產業客戶數量與佔比
- 地區分布：客戶地理位置分布
- 等級分布：A/B/C/D 各級客戶數量

### 2. 行為分析
- 活躍度：近 30/60/90 天有互動的客戶比例
- 回應率：主動聯繫後的回應比率
- 互動頻率：平均每月互動次數

### 3. 價值分析
- 營收貢獻：各客戶的累計營收
- 成長潛力：營收成長率排名
- 推薦價值：透過該客戶轉介的新客數

## 操作步驟

1. 確認分析目標與時間範圍
2. 透過 `cs_customer_db.py` 從 SQLite（`cs_customers.db`）拉取客戶資料、互動記錄、交易數據
3. 依指定維度進行計算
4. 產出結構化分析報告
5. 附上關鍵洞察與行動建議

## 輸出格式

```
客戶分析報告 — 2024 年 Q1

總客戶數：128
A 級：15（12%）/ B 級：38（30%）/ C 級：52（41%）/ D 級：23（18%）

活躍率：68%（87/128）
平均互動頻率：2.3 次/月

最熱門產業：資訊科技（32%）、製造業（18%）
最活躍客戶：ABC 科技（12 次互動）

關鍵洞察：
- B 級客戶中有 8 位互動頻率提升，建議重新評分
- D 級客戶中有 5 位超過 90 天無互動，建議清理
```

## 工具指引

- **cs_customer_db.py**：讀取 SQLite `cs_customers.db` 客戶資料庫、互動記錄、交易數據
  - `get_weekly_stats(brand_id)` — 取得品牌週統計（互動量、活躍客戶數）
  - `get_top_intents(brand_id, limit)` — 取得最常見意圖分布（行為分析）
  - `get_vip_candidates(brand_id, min_conversations, min_positive_ratio)` — 取得高潛力客戶清單
  - `get_customer_history(customer_id, limit)` — 取得個別客戶互動詳情
  - `list_brands()` — 列出所有品牌進行跨品牌比較分析
- **value_scorer**：客戶等級數據
- **interaction_tracker**：互動頻率數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 數據不足 | 說明可用數據量，以現有數據產出部分報告 |
| 時間範圍無資料 | 提示該期間無記錄，建議調整範圍 |
| 計算結果異常 | 標註異常值，建議人工覆核原始數據 |
| SQLite 查詢逾時 | 縮小查詢範圍後重試，確認 `cs_customers.db` 索引是否正常 |

## 使用範例

- 「分析一下我的客戶產業分布」
- 「這季度有多少活躍客戶？」
- 「哪些 B 級客戶最有可能升級為 A 級？」

## 防護規則

- 分析報告僅供內部決策參考，不可對外公開
- 涉及營收的具體金額不可在公開場合提及
- 分析結論需基於數據，不可加入無根據的推測
- 個別客戶的詳細數據需有權限才能查看
