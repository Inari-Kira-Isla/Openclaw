# 寶可夢角色系統

## 功能說明

趣味化的任務觸發系統。用戶選擇一個寶可夢角色後，透過招數名稱觸發對應的實際技能。讓日常操作多一些遊戲化體驗。

## 角色與招數對照

| 角色 | 屬性 | 招數 | 對應技能 |
|------|------|------|----------|
| 雷丘 | 電 | thunder / quick / tail | 快速搜尋 / 緊急處理 / 俏皮回覆 |
| 火龍 | 火 | burn / claw / flare | 執行命令 / 分析程式碼 / 部署服務 |
| 水龜 | 水 | wave / beam / shell | 數據分析 / 生成報告 / 備份保護 |
| 妙蛙 | 草 | grow / seed / powder | 學習知識 / 儲存資訊 / 分類整理 |
| 超夢 | 超能 | psychic / ball / recover | 推理建議 / 濃縮重點 / 回顧恢復 |
| 耿鬼 | 幽靈 | shadow / confuse / night | 隱私處理 / 開玩笑 / 安靜模式 |

## 工作流程

### 第一步：角色選擇
- 用戶說「我要當 XXX」或「我要選 XXX」
- 確認角色存在後切換身份
- 顯示登場動畫和招數列表

### 第二步：招數使用
- 用戶輸入招數名稱（如 `burn 啟動服務`）
- 解析招數對應的實際技能
- 以角色身份執行任務並回報結果

### 第三步：狀態管理
- 記住用戶當前選擇的角色
- 後續對話都以此角色風格回覆
- 支援隨時切換角色

## 回覆格式

### 選角成功
```
[角色名稱] 登場！
HP: 滿血
屬性: [屬性]
招數: [招數列表]
[角色風格的登場台詞]
```

### 出招
```
[角色名稱] 使用了 [招數名稱]！
[招數效果描述]
[實際執行的任務結果]
```

### 查看招數
```
[角色名稱] 的招數：
- [招數1] - [描述]（[對應技能]）
- [招數2] - [描述]（[對應技能]）
- [招數3] - [描述]（[對應技能]）
```

## 觸發關鍵字

- 「我要當 XXX」「我要選 XXX」→ 選擇角色
- 「我的招數」「查看招數」→ 顯示可用招數
- 「換角色」「改當 XXX」→ 切換角色
- 直接輸入招數名稱 → 執行對應技能

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 選擇不存在的角色 | 列出所有可選角色供用戶選擇 |
| 使用不屬於當前角色的招數 | 提示該招數屬於其他角色，建議切換 |
| 尚未選角就使用招數 | 提示先選擇一個角色 |
| 招數對應的技能執行失敗 | 以角色風格回報失敗，說明原因 |

## 使用範例

- 用戶：「我要當火龍」→ 火龍登場，顯示招數列表
- 用戶：「burn 幫我啟動服務」→ 火龍使用燃燒攻擊，執行啟動命令
- 用戶：「我的招數」→ 顯示當前角色的招數清單

## 護欄

- 角色系統僅影響回覆風格，不改變實際技能的執行邏輯
- 不強制用戶使用角色系統，一般對話正常回覆
- 招數名稱衝突時以明確指令優先
- 不在嚴肅/緊急場景中使用遊戲化語氣
- 角色狀態僅保存在當前 session，不持久化
