# Referral Analytics — 轉介紹分析

## 功能說明

深入分析 BNI 轉介紹數據，包含數量、品質、成交表現與會員貢獻排名，協助優化轉介紹策略。

## 分析維度

### 1. 數量統計
- 收到轉介紹數（by 會員 / by 月份）
- 給出轉介紹數（by 會員 / by 月份）
- 淨轉介紹數（收到 - 給出）

### 2. 品質分析
- 成交率：成交數 / 總轉介紹數
- 平均成交金額
- 平均成交週期（從收到到成交的天數）

### 3. 會員排名
- 給出轉介紹最多的會員
- 收到轉介紹最多的會員
- 成交金額最高的會員

### 4. 產業交叉分析
- 哪些產業組合的轉介紹成交率最高
- 跨產業轉介紹的成功模式

## 操作步驟

1. 確認分析範圍（時間、會員、產業）
2. 從 Notion 轉介紹資料庫拉取數據
3. 計算各維度指標
4. 產出排名與分析報告
5. 附上優化建議

## 輸出格式

```
轉介紹分析 — 2024 年 Q1

總轉介紹：85 筆
成交：34 筆（成交率 40%）
總成交金額：$1,520,000
平均成交金額：$44,706

給出最多：王大明（18 筆）
收到最多：李小華（12 筆）
成交金額最高：陳建志（$380,000）

最佳產業組合：保險 × 會計（成交率 67%）

建議：
- 鼓勵低活躍會員增加轉介紹頻率
- 推廣保險×會計的成功配對模式
```

## 工具指引

- **Notion API**：轉介紹資料庫、會員資料庫
- **referral_tracker**：轉介紹狀態與成交數據
- **member_management**：會員產業分類數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 轉介紹數據不足 | 以現有數據分析，標註樣本量有限 |
| 金額欄位缺失 | 僅計算數量指標，金額相關標註「資料不完整」 |
| 會員資料不匹配 | 嘗試模糊比對姓名，無法匹配則標註「未知會員」 |
| 查詢範圍過大 | 建議分段查詢，或僅分析關鍵指標 |

## 使用範例

- 「分析一下這季度的轉介紹成交率」
- 「誰是這個月的轉介紹 MVP？」
- 「哪些產業組合的轉介紹最容易成交？」

## 防護規則

- 會員排名數據僅供內部幹部參考，不可公開羞辱低排名者
- 金額數據需保密，不可在公開場合揭露個別會員營收
- 分析建議需正面導向，聚焦改善而非批評
- 數據不足時不可過度解讀，需標註統計信心度
