# Diagnostic Analysis

## 診斷框架

### 漏斗診斷模型

```
輸入: 多個數據源的指標
輸出: 問題診斷 + 具體建議行動
```

### 經典診斷邏輯（Larry Pattern）

| 指標組合 | 診斷結果 | 建議行動 |
|----------|----------|----------|
| 觀看高 + 下載低 | Hook 吸引錯受眾 | 更換 CTA 或調整目標受眾 |
| 下載高 + 付費低 | App onboarding 問題 | 檢查轉化流程 |
| 觀看低 + 轉換高 | 需要更強 hook 拉流量 | 優化 hook |
| 曝光高 + 點擊低 | 素材不夠吸引 | 優化視覺或標題 |
| 點擊高 + 轉化低 | 著陸頁問題 | 檢查 Landing Page |

---

## 診斷流程

### 1. 數據收集
- 收集多個相關指標
- 確保數據時間範圍一致
- 識別數據來源

### 2. 模式識別
- 對照診斷邏輯表
- 識別異常模式
- 計算關鍵比率

### 3. 問題定位
- 確定問題環節
- 評估嚴重程度
- 優先級排序

### 4. 建議生成
- 給出具體可執行的建議
- 說明原因
- 提供參考案例（如有）

## 輸出格式

### 診斷報告結構

```markdown
## 📊 診斷摘要
[一句話說明發現的問題]

## 🔍 數據分析
| 指標 | 數值 | 狀態 |
|------|------|------|
| xxx  | xxx  | ✅/⚠️/❌ |

## 🎯 問題診斷
### 主問題
[問題描述]

### 根因分析
[深入分析]

## 💡 建議行動
### [優先級] 立即行動
- [具體建議]
- [預期效果]

### [優先級] 短期優化
- [優化方向]

### [優先級] 長期改進
- [戰略調整]
```

- 「分析一下為什麼下載量高但付費轉化低」

- 「我的課程銷售漏斗哪裡有問題」
- 「分析這週營收下降的原因」

## 擴展診斷邏輯

可以根據業務類型擴展診斷規則：

### 電商
- 瀏覽高 + 加入購物車低 → 產品頁優化
- 加入購物車高 + 結帳低 → 結帳流程優化

### SaaS
- 註冊高 + 激活低 → 註冊流程優化
- 激活高 + 付費低 → 試用期價值傳遞問題

### 內容變現
- 觀看高 + 訂閱低 → 內容與變現路徑斷裂
