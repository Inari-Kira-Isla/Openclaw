# Diagnostic Analysis — 診斷分析

## 功能說明

針對業務數據中的異常或瓶頸進行深入診斷，透過漏斗模型與指標交叉分析定位問題根因，產出具體可執行的改善建議。

## 診斷框架

### 漏斗診斷模型

```
輸入: 多個數據源的指標
↓ 交叉比對
輸出: 問題定位 + 根因分析 + 建議行動
```

### 常見診斷邏輯

| 指標組合 | 診斷結果 | 建議行動 |
|----------|----------|----------|
| 曝光高 + 點擊低 | 素材不夠吸引 | 優化視覺或標題 |
| 點擊高 + 轉化低 | 著陸頁問題 | 檢查 Landing Page 體驗 |
| 觀看高 + 下載低 | Hook 吸引錯受眾 | 調整目標受眾或 CTA |
| 下載高 + 付費低 | Onboarding 不佳 | 優化新手引導流程 |
| 瀏覽高 + 加購低 | 產品頁轉化差 | 優化產品描述、圖片、定價 |
| 加購高 + 結帳低 | 結帳流程有阻力 | 簡化結帳步驟 |

### 業務類型擴展

| 業務類型 | 漏斗階段 |
|----------|----------|
| 電商 | 瀏覽 → 加入購物車 → 結帳 → 成交 |
| SaaS | 註冊 → 激活 → 使用 → 付費 |
| 內容變現 | 觀看 → 訂閱 → 付費 → 續費 |
| BNI | 接觸 → 轉介紹 → 報價 → 成交 |

## 操作步驟

1. 收集使用者描述的問題或異常現象
2. 確認相關指標與數據來源
3. 拉取多個指標的數據
4. 交叉比對，對照診斷邏輯表
5. 定位問題環節，評估嚴重程度
6. 產出診斷報告與行動建議

## 輸出格式

```
診斷報告

診斷摘要：TikTok 廣告觀看量高但下載率僅 2%，問題在於 CTA 與受眾不匹配

數據分析：
| 指標     | 數值    | 狀態 |
|----------|---------|------|
| 觀看數   | 50,000  | 正常 |
| 點擊率   | 3.2%    | 正常 |
| 下載率   | 0.8%    | 偏低 |
| 付費率   | 15%     | 正常 |

問題定位：點擊→下載 環節（轉化率 0.8%，基準應 > 3%）

根因分析：
Hook 吸引的受眾對產品實際功能無需求，導致點擊後不下載

建議行動：
1.【立即】更換 CTA，明確說明產品用途
2.【短期】調整廣告受眾定向
3.【長期】測試不同 Hook 類型找出高轉化素材
```

## 工具指引

- **本地資料庫（SQLite）**：業務數據來源
- **trend_analysis**：趨勢數據輔助診斷
- **performance_report**：KPI 數據基準

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 指標數據不足 | 說明缺少哪些數據，以現有指標做有限診斷 |
| 無法匹配診斷邏輯 | 告知使用者情況較複雜，建議人工深入分析 |
| 數據品質差 | 標註數據可信度，建議先清理數據 |
| 多重問題交疊 | 按嚴重程度排序，逐一診斷 |

## 使用範例

- 「分析一下為什麼下載量高但付費轉化低」
- 「診斷我的 TikTok 流量為什麼轉換不好」
- 「我的課程銷售漏斗哪裡有問題」
- 「分析這週營收下降的原因」

## 防護規則

- 診斷結論需基於數據，不可無根據推測
- 建議行動需具體可執行，不可過於抽象
- 當數據不足以做出明確診斷時，需坦誠說明限制
- 不可將診斷結果歸咎於特定個人
- 涉及外部平台（TikTok、Facebook）的分析需提醒平台政策可能影響
