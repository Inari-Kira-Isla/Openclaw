# Trend Analysis — 趨勢分析

## 功能說明

對業務數據進行時間維度的分析，識別成長趨勢、季節性波動與異常訊號，並提供簡單的趨勢預測與行動建議。

## 分析方法

### 1. 時間序列分析
- 移動平均：消除短期波動，顯示長期趨勢
- 季節性調整：識別週期性模式（月度、季度）
- 異常檢測：標註偏離正常範圍的數據點

### 2. 對比分析
- 環比：本期 vs 上期（月對月、週對週）
- 同比：本期 vs 去年同期
- 目標對比：實際值 vs 目標值

### 3. 趨勢預測
- 線性回歸：基於歷史數據的線性推估
- 成長率預測：依近期成長率推算未來值

## 操作步驟

1. 確認分析指標（營收、轉介紹數、客戶數等）
2. 確認時間範圍與對比基準
3. 從本地資料庫（SQLite）拉取歷史數據
4. 執行趨勢分析計算
5. 標註關鍵轉折點與異常值
6. 輸出分析報告與預測建議

## 輸出格式

```
趨勢分析 — 月營收（近 6 個月）

月份    | 營收      | 環比     | 同比
1 月    | $280,000  | +5%     | +15%
2 月    | $310,000  | +11%    | +22%
3 月    | $295,000  | -5%     | +18%

趨勢：整體上升，2 月為近期高點
預測下月：$305,000（基於 3 個月移動平均）

建議：
1. 2 月成長突出，分析成功因素並複製
2. 3 月小幅回落，屬正常波動，無需擔憂
```

## 工具指引

- **本地資料庫（SQLite）**：讀取業務歷史數據
- **performance_report**：績效數據來源
- **referral_analytics**：轉介紹趨勢數據

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 歷史數據不足（< 3 個月） | 僅做環比分析，不做預測，標註數據量不足 |
| 數據斷層（某月無記錄） | 標註缺失，以前後平均值補插，註明為估算 |
| 異常值干擾趨勢 | 標註異常值，提供含/不含異常值的兩組分析 |
| 指標定義不明 | 向使用者確認具體指標 |

## 使用範例

- 「分析一下近半年的營收趨勢」
- 「轉介紹數量有在成長嗎？」
- 「預測下個月的客戶數大概多少？」

## 防護規則

- 預測值需標註「估算」，不可作為承諾
- 數據不足時不可強行預測，需告知限制
- 趨勢分析需客觀呈現，不可只報喜不報憂
- 營收等敏感數據的趨勢報告僅限內部使用
