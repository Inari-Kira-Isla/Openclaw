# Question Analytics — 問題分析

## 功能說明

分析客戶在各管道（Facebook Messenger、Telegram 等）提出的問題，找出熱門問題、回覆效果與趨勢變化，用於優化 FAQ 與客服流程。

## 分析維度

### 1. 問題分布
- 各類型問題佔比（價格 / 產品 / 服務 / 投訴 / 其他）
- 熱門問題 Top 10 排名
- 新出現的問題類型標記

### 2. 回覆效果
- 自動回覆率：由 AI/FAQ 成功處理的比例
- 轉人工率：需轉接人工的比例
- 客戶滿意度：回覆後客戶是否持續追問

### 3. 趨勢變化
- 週度 / 月度問題量變化
- 季節性波動分析
- 新問題類型增長趨勢

## 操作步驟

1. 確認分析範圍（時間、管道、問題類型）
2. 從本地資料庫（SQLite）對話記錄與分類數據中拉取原始資料
3. 計算各維度指標
4. 產出分析報告與可視化摘要
5. 附上優化建議

## 輸出格式

```
問題分析報告 — 2024 年 1 月

總問題數：156
自動回覆率：72%（112/156）
轉人工率：28%（44/156）

熱門問題：
1. 價格詢問 — 35%（55 次）
2. 服務範圍 — 25%（39 次）
3. 產品功能 — 20%（31 次）

平均回覆時間：3.2 分鐘
問題解決率：85%

建議：「運費」相關問題本月增加 40%，建議新增 FAQ 模板
```

## 工具指引

- **本地資料庫（SQLite）**：讀取對話記錄、問題分類數據
- **conversation_logger**：對話歷史來源
- **question_classifier**：分類標籤來源

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 分類數據不完整 | 標註「部分資料未分類」，以已分類數據分析 |
| 時間範圍內無資料 | 提示無記錄，建議調整範圍 |
| 計算異常 | 標註異常項，建議人工覆核 |
| 資料庫查詢逾時 | 縮小查詢範圍後重試 |

## 使用範例

- 「分析一下這個月客戶最常問什麼問題」
- 「自動回覆的成功率是多少？」
- 「跟上個月比，投訴類問題有增加嗎？」

## 防護規則

- 分析結果不含個別客戶的私人資訊
- 報告中的具體對話內容需脫敏處理
- 分析結論需基於數據，不可主觀臆測
- 不可將分析報告直接發送給外部人員
