# Requirement Analyzer

## 分析流程

```
需求輸入 → 理解 → 提取 → 評估 → 輸出
```

## 需求類型

### 功能需求
- 核心功能
- 輔助功能
- 擴展功能

### 非功能需求
- 效能要求
- 安全要求
- 可用性要求
- 相容性要求

### 約束條件
- 技術限制
- 資源限制
- 時間限制

## 複雜度評估

### 評估維度
| 維度 | 低 | 中 | 高 |
|------|----|----|-----|
| 功能數 | 1-3 | 4-6 | 7+ |
| 整合度 | 無 | 簡單 | 複雜 |
| 特殊需求 | 無 | 少 | 多 |

### 複雜度分級
- **Simple**: 單一功能，無整合
- **Medium**: 2-3 功能，簡單整合
- **Complex**: 多功能，複雜整合

## 輸出格式

```json
{
  "requirements": {
    "functional": ["功能1", "功能2"],
    "non_functional": ["效能", "安全"],
    "constraints": ["限制1"]
  },
  "complexity": "medium",
  "estimated_effort": "2-4 小時",
  "risks": ["風險1", "風險2"],
  "clarifications_needed": ["問題1"]
}
```

## 優先級
| 等級 | 標準 |
|------|------|
| P0 | 核心功能，必需 |
| P1 | 重要功能，強烈建議 |
| P2 | 輔助功能，可選 |
