# Expert Skill MD

## SKILL.md 結構

### 必須包含

1. **YAML Frontmatter**
   ```yaml
   ---
   name: skill-name
   description: 技能描述（觸發條件）
   ---
   ```

2. **主要內容**
   - 功能說明
   - 使用場景
   - 參數說明
   - 範例

### 建議章節

- ## 概述 - 簡短介紹
- ## 參數 - 輸入參數說明
- ## 輸出 - 輸出格式說明
- ## 範例 - 使用範例
- ## 限制 - 已知限制

## 寫作原則

### 1. 簡潔明確
- 每句話都要有價值
- 避免冗餘解釋
- 使用主動語態

### 2. 範例豐富
- 包含正常情況範例
- 包含邊界情況範例
- 包含錯誤處理範例

### 3. 結構清晰
- 使用標題層次
- 適當使用列表
- 表格呈現結構化資訊

## 描述撰寫指南

### 觸發條件格式
```
[技能名稱] - [功能簡述]。當 [何時觸發] 時使用，包括：[包含功能]。
```

範例：
```
天氣查詢 - 取得指定地點的天氣資訊。當用戶詢問天氣、溫度、降雨機率時觸發，包括：目前天氣、預報查詢、氣象警告。
```

### 避免的寫法
- ❌ 過於籠統：「可以做很多事」
- ❌ 過於技術：「使用 XXX 框架的 YYY 方法」
- ❌ 缺少範例：「如何使用見下文」

## 質量檢查清單

- [ ] name 符合命名規範（小寫、連字符）
- [ ] description 完整描述功能與觸發條件
- [ ] 包含實際可用的範例
- [ ] 沒有語法錯誤
- [ ] Markdown 格式正確
