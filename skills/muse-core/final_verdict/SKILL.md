# 最終裁決

對多個 Agent 的輸出進行評估、衝突解決與質量判斷，產出最終結果。

## 操作 / 工作流程

1. **收集結果** — 從各 Agent 收集所有輸出，確認資料完整
2. **個別評估** — 依據準確性、完整性、一致性、可用性四維度評分
3. **衝突檢測** — 比對結果，找出結論矛盾、資料衝突、風格不一致
4. **衝突解決** — 依下表策略處理：
   | 衝突類型 | 解決策略 |
   |----------|----------|
   | 結論矛盾 | 採用更高優先級 Agent 的結果 |
   | 資料衝突 | 採用最新來源 |
   | 風格不一致 | 統一為標準格式 |
   | 遺漏項目 | 補充缺失內容 |
5. **最終輸出** — 整合為用戶可直接使用的回覆

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| agent_results | array | — | 各 Agent 的輸出結果列表 |
| quality_threshold | number | 70 | 質量分數門檻（0-100） |
| conflict_strategy | string | priority | 衝突解決策略：priority / newest / merge |

## 輸出格式

```
⚖️ 裁決結果：[approved / needs_revision / rejected]

📊 質量分數：[0-100] / 信心度：[0-1]

📝 摘要：
[整合後的最終輸出]

⚠️ 需修改項目：（若有）
- [修改項目]
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| Agent 結果為空 | 回報缺失的 Agent，請求重新執行 |
| 所有結果品質低於門檻 | 標記 rejected，列出各項不足 |
| 衝突無法自動解決 | 列出衝突細節，請求人工裁決 |

## 使用範例

- "幫我整合所有 agent 的結果，做最終判斷"
- "這幾個結果有矛盾，幫我裁決"
- "評估一下這些輸出的品質"
