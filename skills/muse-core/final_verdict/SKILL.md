# Final Verdict

## 核心功能

1. **結果評估** - 評估各 Agent 輸出的質量
2. **衝突解決** - 處理多個結果的矛盾
3. **質量判斷** - 判斷是否達標
4. **最終輸出** - 生成用戶可用的回覆

## 評估維度

### 準確性
- 事實是否正確
- 邏輯是否合理
- 數據是否準確

### 完整性
- 是否覆蓋所有需求
- 是否有遺漏
- 邊界情況是否處理

### 一致性
- 多個結果是否一致
- 風格是否統一
- 格式是否規範

### 可用性
- 用戶是否能直接使用
- 是否需要額外處理
- 呈現是否清晰

## 裁決流程

```
收集結果 → 個別評估 → 衝突檢測 → 整合方案 → 最終輸出
```

### 衝突解決策略

| 衝突類型 | 解決策略 |
|----------|----------|
| 結論矛盾 | 採用更高優先級 Agent 的結果 |
| 資料衝突 | 採用更新的來源 |
| 風格不一致 | 統一為標準格式 |
| 遺漏項目 | 補充缺失內容 |

## 輸出格式

```json
{
  "verdict": "approved|needs_revision|rejected",
  "summary": "結果摘要",
  "quality_score": 0-100,
  "integrated_output": "整合後的最終輸出",
  "revision_needed": ["需要修改的項目"],
  "confidence": 0-1
}
```
