# 技能融合建議

分析多個相似或重疊的技能，提出融合方案以降低複雜度。

## 操作 / 工作流程

1. **重疊分析** — 讀取目標技能的 SKILL.md，由 LLM 比對：
   - 功能重疊度（相同觸發條件、相同操作）
   - 架構相似度（輸入輸出格式、工具依賴）
   - 使用頻率（透過 `memory_search` 查詢使用紀錄）
2. **方案設計** — 依重疊程度建議融合方式：
   - 高重疊（> 70%）→ **合併**：兩者合為一個新技能
   - 中重疊（40-70%）→ **整合**：一主一輔，擴展主技能
   - 低重疊（< 40%）→ **鏈結**：保持獨立，建立呼叫鏈
3. **影響評估** — 分析融合對現有系統的影響：
   - 哪些 agent 會受影響
   - 是否需要遷移配置
   - 有無破壞性變更
4. **產出提案** — 生成完整融合提案供用戶審核

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| skills | array | 必填 | 要比較的技能名稱列表（至少 2 個） |
| agent | string | "" | 所屬 agent，空則自動偵測 |

## 輸出格式

```
🧬 技能融合提案
- 比較技能：{skill_a} vs {skill_b}
- 功能重疊度：{overlap}%
- 建議方案：{merge / integrate / link}

方案詳情：
{proposal_detail}

影響範圍：
- 受影響 Agent：{agents}
- 遷移步驟：{steps}
- 風險：{risks}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 技能不存在 | 列出可用技能，請用戶重新選擇 |
| 僅提供 1 個技能 | 自動搜尋相似技能進行比較 |
| 無重疊（< 10%） | 回報「無需融合」，結束流程 |
| 融合影響過大 | 標記為高風險，建議分階段執行 |

## 使用範例

- "conflict_detection 和 decay_monitor 需要合併嗎"
- "分析一下這兩個技能的重疊度"
- "建議哪些技能可以整合"
