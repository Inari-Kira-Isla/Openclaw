# 技能演進管理

分析技能的使用狀況與效能，識別改進機會並規劃演進路線。

## 操作 / 工作流程

1. **數據收集** — 透過 `memory_search` 取得技能使用紀錄：
   - 觸發頻率
   - 成功率 / 失敗率
   - 平均執行時間
   - 用戶回饋
2. **問題識別** — LLM 分析數據，找出：
   - 常見失敗模式
   - 未覆蓋的邊界情況
   - 效能瓶頸
   - 功能缺口
3. **演進分類** — 將改進項目分類：
   - 功能增強：新增能力、擴展範圍
   - 效能優化：減少延遲、節省資源
   - 架構重構：模組化、標準化
   - 裁剪精簡：移除冗餘、聚焦核心
4. **規劃排序** — 依優先級排序：
   - 低風險改動先行
   - 用戶影響大的優先
   - 核心功能優先
5. **產出演進計畫** — 記錄到 `memory_search`，追蹤執行進度

## 參數

| 參數 | 類型 | 預設 | 說明 |
|------|------|------|------|
| skill_name | string | 必填 | 要分析的技能名稱 |
| period | string | "30d" | 分析時間範圍 |
| action | string | "analyze" | 動作：analyze / plan / track |

## 輸出格式

```
🦋 技能演進報告 — {skill_name}
- 分析期間：{period}
- 觸發次數：{trigger_count}
- 成功率：{success_rate}%
- 平均執行時間：{avg_time}

改進建議（依優先級）：
1. [{type}] {description} — 預估影響：{impact}
2. [{type}] {description} — 預估影響：{impact}

下一步：{next_action}
```

## 錯誤處理

| 錯誤 | 處理 |
|------|------|
| 無使用紀錄 | 回報「無足夠數據」，建議累積使用後再分析 |
| 技能不存在 | 列出可用技能，請用戶確認名稱 |
| 分析期間過短（< 7 天） | 警告數據量不足，結果可能不準確 |
| 演進計畫衝突 | 與現有計畫比對，合併或調整優先級 |

## 使用範例

- "分析 semantic_ingest 最近的表現"
- "哪些技能需要升級"
- "規劃 memory_deepener 的下一步演進"
