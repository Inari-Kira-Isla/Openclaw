# Rule Enforcement

## 功能說明

確保系統中所有 agent 與操作遵守既定規則和政策。負責違規偵測、處罰執行與申訴處理，維護系統治理的一致性與公正性。

## 規則類型

| 類型 | 涵蓋範圍 | 範例 |
|------|----------|------|
| 安全規則 | 認證、權限、資料保護 | 禁止未授權存取、敏感資料必須加密 |
| 行為規則 | 使用規範、內容規範 | 禁止生成有害內容、回覆需遵守格式 |
| 業務規則 | 流程規範、審批要求 | 高風險操作需核准、支出超額需審批 |

## 執行步驟

### 1. 規則比對

- 接收待檢查的操作或行為
- 查找適用的規則清單
- 逐條比對是否符合規則

### 2. 違規偵測

- 若發現不符合項目，記錄違規事實
- 判斷違規嚴重程度
- 收集相關證據（操作記錄、時間戳）

### 3. 處罰執行

依違規嚴重程度執行對應處罰：

| 違規等級 | 處罰方式 | 說明 |
|----------|----------|------|
| 輕微 | 警告 | 發出警告通知，記錄在案 |
| 中等 | 限制權限 | 暫時限制相關功能的使用 |
| 嚴重 | 暫停帳號 | 暫停 agent 或功能運作 |
| 重大 | 終止帳號 | 永久停用並通報管理者 |

### 4. 記錄與通報

- 記錄違規事件詳情
- 通知相關人員或管理者
- 更新違規紀錄檔

### 5. 申訴處理

- 接收申訴請求
- 複核違規證據與處罰決定
- 維持原判或調整處罰

## 執行流程圖

```
規則觸發 -> 違規檢測 -> 嚴重程度判定 -> 警告/處罰 -> 記錄 -> 通報
                                                          |
                                                      申訴處理 -> 覆核 -> 維持/調整
```

## 工具指令

- 本技能為治理邏輯，不依賴外部 API
- 與 risk_assessment 配合進行風險等級判斷
- 與 conflict_resolution 配合處理爭議案件

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 規則定義模糊 | 偏向寬鬆解釋，但記錄待釐清並建議更新規則 |
| 證據不足 | 不執行處罰，僅發出觀察通知 |
| 處罰等級爭議 | 提交 conflict_resolution 仲裁 |
| 同一行為多條規則適用 | 取最嚴格的規則適用 |
| 申訴內容合理 | 撤銷或降級處罰，記錄修正原因 |

## 使用範例

- 「檢查這個操作是否符合安全規則」
- 「Agent X 違反了使用規範，需要處理」
- 「我要對上次的警告提出申訴」

## 安全邊界

- 處罰前必須有明確的違規事實與證據
- 不可因單一事件直接執行「重大」等級處罰（除非涉及安全漏洞）
- 申訴期間不得加重處罰
- 規則變更不具追溯效力（不以新規則懲罰舊行為）
- 所有處罰決定必須記錄，可供事後審計
- 不得自行修改規則，規則變更需管理者授權
