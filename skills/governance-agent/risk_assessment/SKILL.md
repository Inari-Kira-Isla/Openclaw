# Risk Assessment

## 功能說明

對系統中的任務、操作或決策進行風險評估，識別潛在風險因素，依據影響程度與發生可能性進行分級，並提出緩解建議。用於輔助 governance-agent 的治理決策。

## 風險類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 操作風險 | 執行過程中可能的錯誤或故障 | 系統故障、執行錯誤、人為失誤 |
| 安全風險 | 資料或系統安全相關威脅 | 資料外洩、未授權存取、惡意攻擊 |
| 業務風險 | 對業務運營的負面影響 | 客戶流失、營收下降、聲譽受損 |

## 執行步驟

### 1. 識別風險因素

- 分析目標任務/操作的內容與範圍
- 列出所有可能的風險因素
- 標記每個風險的類型（操作/安全/業務）

### 2. 量化評估

使用「影響 x 可能性」矩陣進行評估：

| 影響 \ 可能性 | 低 | 中 | 高 |
|---------------|------|------|--------|
| **高** | 中風險 | 高風險 | 嚴重 |
| **中** | 低風險 | 中風險 | 高風險 |
| **低** | 低風險 | 低風險 | 中風險 |

### 3. 決定風險等級

| 等級 | 行動 |
|------|------|
| 低風險 | 可直接執行，記錄備查 |
| 中風險 | 需額外確認後執行，設定監控 |
| 高風險 | 需管理者核准，制定緩解計畫 |
| 嚴重 | 暫停執行，立即升級處理 |

### 4. 輸出評估結果

```json
{
  "risk_level": "high",
  "risk_type": "安全風險",
  "factors": ["因素1", "因素2"],
  "impact": "high",
  "likelihood": "medium",
  "mitigation": ["緩解措施1", "緩解措施2"],
  "recommendation": "建議核准後執行，並設定即時監控"
}
```

## 工具指令

- 本技能為純邏輯判斷，不依賴外部工具
- 評估結果可供其他 governance 技能（rule_enforcement、conflict_resolution）引用

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 資訊不足以評估 | 列出需要補充的資訊，暫不給出最終等級 |
| 風險因素模糊 | 假設最壞情況，偏向高估風險（寧嚴勿鬆） |
| 多重風險疊加 | 取最高風險等級，並標注複合風險 |
| 評估結果被質疑 | 提供評估依據與推理過程供覆核 |

## 使用範例

- 「評估這個 API 變更的風險」
- 「刪除這批資料有什麼風險？」
- 「新功能上線前的風險分析」

## 安全邊界

- 風險評估僅供參考，最終決策權在管理者
- 不可因風險評估結果自行決定阻止或執行任務
- 嚴重風險必須升級通報，不得靜默處理
- 評估過程中不得存取或修改被評估對象的資料
- 保持客觀中立，不因過往結果偏移評估標準
