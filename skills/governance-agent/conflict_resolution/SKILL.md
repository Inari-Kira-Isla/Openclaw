# Conflict Resolution

## 功能說明

解決系統運作中出現的各類衝突，包括資源爭用、規則矛盾、優先級分歧等。透過協商、仲裁或升級機制，找出公正且可執行的解決方案。

## 衝突類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 資源衝突 | 多方同時需要同一資源 | 兩個 agent 同時存取同一資料庫 |
| 規則衝突 | 規則之間互相矛盾 | 安全規則與效率規則衝突 |
| 優先級衝突 | 任務優先順序分歧 | 多項緊急任務同時需要處理 |
| 決策衝突 | 多方意見不一致 | 不同 agent 對同一問題有不同判斷 |

## 執行步驟

### 1. 衝突識別

- 接收衝突報告或自動偵測衝突
- 確認衝突的類型與涉及方
- 收集各方立場與相關證據

### 2. 方案評估

針對衝突類型選擇解決策略：

| 策略 | 適用情境 | 方法 |
|------|----------|------|
| 協商 | 各方有彈性空間 | 找共同點，雙方讓步達成共識 |
| 仲裁 | 有明確規則可依循 | 根據規則做出裁決 |
| 優先級排序 | 資源/時間衝突 | 依緊急度與重要性排定順序 |
| 升級處理 | 超出自動化能力 | 提交管理者介入決定 |

### 3. 執行解決方案

- 通知相關各方決議內容
- 確認各方同意執行（協商）或告知裁決結果（仲裁）
- 執行資源重新分配或優先級調整

### 4. 記錄結果

```json
{
  "conflict_type": "資源衝突",
  "parties": ["agent_A", "agent_B"],
  "description": "衝突描述",
  "strategy": "仲裁",
  "resolution": "解決方案內容",
  "parties_agreed": true,
  "next_steps": ["後續步驟1", "後續步驟2"]
}
```

## 工具指令

- 本技能為治理邏輯，不依賴外部 API
- 與 risk_assessment 配合評估各方案的風險
- 與 rule_enforcement 配合檢查方案是否符合規則

## 錯誤處理

| 情境 | 處理方式 |
|------|----------|
| 各方拒絕協商 | 升級為仲裁模式，依規則裁決 |
| 無明確規則可依循 | 依「最小傷害原則」裁決，並建議制定新規則 |
| 衝突持續無法解決 | 升級至管理者人工介入 |
| 裁決後一方不服 | 允許申訴，由 rule_enforcement 處理 |
| 衝突涉及安全問題 | 優先確保安全，暫停衝突雙方的操作 |

## 使用範例

- 「Agent A 和 Agent B 同時需要寫入同一個資料表」
- 「這兩條規則互相矛盾，應該以哪個為準？」
- 「三個任務都是緊急的，怎麼排優先？」

## 安全邊界

- 解決方案不得違反任何安全規則
- 仲裁結果必須有明確依據，不可武斷裁決
- 不偏袒任何一方，保持中立立場
- 涉及安全風險的衝突一律採取保守策略（優先保安全）
- 所有裁決記錄必須保留，可供事後審計
- 不得在未通知各方的情況下執行單方面決議
