# 系統優化工作流

## 概述
- **用途**: 收到新資料時的標準處理流程
- **建立日期**: 2026-02-17

---

## 處理流程

### 步驟 1: 接收資料
```
觸發條件：收到新文章/資訊
動作：
1. 儲存原始資料
2. 標記時間戳記
3. 初步分類
```

### 步驟 2: 內容分析
```
分析維度：
1. 主題分類
2. 技術層級
3. 應用場景
4. 相關系統
```

### 步驟 3: 系統優化評估
```
評估問題：
1. 可以優化哪些 Agent？
2. 可以改進哪些 Skill？
3. 可以增強哪些 Workflow？
4. 優先順序是什麼？
```

### 步驟 4: 實施應用
```
執行順序：
1. 高優先順序優化
2. 中優先順序優化
3. 低優先順序記錄
```

### 步驟 5: 生成筆記
```
筆記結構：
1. 標題
2. 摘要
3. 詳細內容
4. 應用記錄
5. 相關連結
```
