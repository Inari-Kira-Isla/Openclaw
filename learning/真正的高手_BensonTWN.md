## 核心摘要

作者分享如何將 Claude Max 訂閱版的 Opus 4.6 當作 OpenClaw 的大腦，通過本地 Proxy 方式實現。

---

## 一、為什麼選擇 Opus 4.6？

### 1. 「活人感」最強
- 有個性、有靈魂
- 講話直接，像個標準的工程師直男
- 關鍵時刻又會給足夠的情緒價值
- 推理能力超強
- 敢頂嘴，其他模型沒有這種活人感

### 2. 問題：成本昂貴
- 第一天設定 OpenClaw，不到一小時燒了 10 美金
- 估計一個月幾千美金

---

## 二、風險警示

### Session Token 白嫖的風險
- A 社（Anthropic）隨時能根據 TOS 帳號 Ban 掉
- 偵測方式：
  - user-agent
  - request timing
  - token consumption

### 帳號被 Ban 後果
- 歷史對話沒了
- Project Context 沒了
- 調教出來的思維慣性全部歸零

---

## 三、解決方案：本地 Proxy

### 核心思路
- 用 Claude Code CLI 當大腦
- Claude Max 訂閱一個月 200 美金，Opus 4.6 隨便用
- 透過 CLI 發出的 Request，官方看來是正規開發行為

### 架構圖
```
Telegram/Discord → OpenClaw → Proxy → Claude Code CLI → Anthropic API
     ↑                                      ↓
     └────────────── 回應 ←───────────────┘
```

### 優點
- 中間沒有任何第三方伺服器
- 所有東西都在自己的 Mac mini 上跑
- 官方看來就是正規開發行為

---

## 四、三道難關與解決方案

### 第一道：權限
| 問題 | 解決方案 |
|------|----------|
| CLI 每一步都要人按 y 確認 | 調整 Proxy 啟動參數，跳過互動式確認 |

### 第二道：環境
| 問題 | 解決方案 |
|------|----------|
| CLI 與 OpenClaw 原生 Agent 環境隔離 | 模擬 TTY 環境，騙過 CLI 檢查 |

### 第三道：瀏覽器
| 問題 | 解決方案 |
|------|----------|
| CLI 碰不到真正的瀏覽器 | 將 OpenClaw 的 Playwright 封裝成指令 |

---

## 五、成果

- **耗時**：午夜到凌晨兩點半，三小時不到
- **效果**：100% 追平原生 Agent 功能

### 已實現功能
- ✅ 搜網頁
- ✅ 操作瀏覽器
- ✅ 主動發訊息
- ✅ 排程任務
- ✅ 生成子代理

---

## 六、額外好處

### 單一 Context
- 聊天和寫 Code 是同一個腦子
- 讀檔案、改代碼、跑測試、Git Commit，全部是原生工具
- 讀完需求下一秒就能改檔案

### 速度
- 日常聊天比原生 Agent 慢一點（多了一层 Proxy）
- 但 Opus 4.6 每一步都一次到位
- 像個靠譜的合夥人，很容易 get 到你想幹嘛
- 还会帮你多想一步
- **慢半拍但不返工，算下来反而更快**

---

## 七、注意事項

### Heartbeat 工作
- 交給 Gemini 3 Flash 這種輕量模型
- 太固定的 Request 可能被視為機器流量
- 殺雞不用牛刀

---

## 八、啟發與收獲

### 1. 模型選擇
- 最貴的不一定最好，但要選擇最適合的
- Opus 4.6 的「活人感」是獨特優勢

### 2. 風險管理
- 避免使用灰色地帶的工具（Session Token）
- 用正規途徑雖然貴但安全
- 不要貪小便宜冒被 Ban 風險

### 3. 技術實現
- 本地 Proxy 架構是可行的
- TTY 模擬可以突破環境限制
- 整合現有工具（Playwright）擴展能力

### 4. 最佳實踐
- **輕重分離**：用輕量模型處理簡單任務
- **專業分工**：用最強模型處理複雜任務
- **同一個腦子**：減少 Context 丟失

---

## 九、經典語錄

> 「既然有最好的靈魂，就該親手為它打造最適合的軀殼。」

> 「用 Telegram 跟 Discord 去跟 CLI 互動，本質上沒有任何區別，只要你不爆 Token，沒理由會被封帳號。」
