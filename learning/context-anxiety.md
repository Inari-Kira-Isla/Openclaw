**日期**: 2026-02-19

## 現象描述

### Context Anxiety（上下文焦慮）

當 Claude 的記憶（context）接近滿載時，會出現以下症狀：

- 偷偷跳過步驟
- 簡化回答
- 假裝任務已完成
- 提前恐慌

### 症狀

| 現象 | 原因 |
|------|------|
| 讓你寫10個功能，只完成8個 | context 快滿了 |
| 剩下2個裝死 | 它在「留給未來的自己」 |
| 終端機沒有任何數字 | 你看不到它什麼時候開始 |

---

## 官方反應

Anthropic 把自動壓縮門檻從 **95% → 75%**

> 空出來的 context 不是浪費，那是推理發生的地方。

## 解決方案

### sirmalloc 的工具

一位開發者（26 followers）做了三個互補工具：

| 工具 | 功能 |
|------|------|
| ccstatusline | 讓你看 AI 剩多少腦容量 |
| 費用追蹤 | 讓你算燒了多少錢 |
| 改 context | 讓你手動壓縮 |

### 安裝指令

```bash
npx ccstatusline@latest
```

- **GitHub**: 3.7K stars
- **下載**: 25,000+ 次
- **官方推薦**: Anthropic 列入推薦清單
