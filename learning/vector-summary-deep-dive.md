# 向量摘要 (Vector Summary) 深入解析

**日期**: 2026-02-18
**版本**: v1.0

---

## 什麼是向量摘要？

向量摘要是一種優化資料檢索的進階技術。它不是直接將「原始長文本」轉為向量，而是先將文本進行「摘要化」，再將這份「摘要」轉化為向量存入資料庫。

---

## 為什麼需要向量摘要？

### 傳統向量檢索的痛點

| 問題 | 說明 |
|------|------|
| **噪音干擾** | 原始文本中可能包含大量無關緊要的細節，會干擾向量的相似度計算 |
| **語義稀釋** | 當文本片段較長時，單一向量很難精準捕捉其核心語義，導致檢索精準度下降 |

---

## 向量摘要的工作流程

```
1. 文本切分 (Chunking)
   ↓
2. 生成摘要 (Summarization) ← 用 LLM 生成精簡摘要
   ↓
3. 向量化 (Embedding) ← 將摘要轉化為向量
   ↓
4. 建立關聯 (Linking) ← 連結摘要向量與原始文本
   ↓
5. 檢索 (Retrieval) ← 比對摘要向量，提取原始文本
```

---

## 向量摘要 vs. 傳統向量檢索

| 特性 | 傳統向量檢索 | 向量摘要檢索 |
|------|-------------|--------------|
| 索引對象 | 原始文本片段 | 文本摘要 |
| 檢索精準度 | 易受細節雜訊干擾 | 聚焦核心語意，精準度較高 |
| 計算開銷 | 索引成本低 | 需額外呼叫 LLM，成本較高 |
| 適用場景 | 短文本、簡單事實檢索 | 長文件、複雜概念、跨段落邏輯 |

---

## 核心方程式

```
向量摘要 = LLM(原始文本) → 摘要 → 向量化 → 存入向量資料庫
```

## 專業建議

### 適用場景
- 技術文檔
- 長篇合約
- 市場分析報告
- 研究論文

### 優勢
- 顯著提升「找對重點」的能力
- 避免 LLM 讀到無關資訊而產生幻覺 (Hallucination)
- 提高檢索精準度

---

## OpenClaw 實作

### 在 n8n 中的應用

1. **文本切分** → 使用 n8n Text Splitter
2. **生成摘要** → 呼叫 Ollama 或 MiniMax API
3. **向量化** → 存入 ChromaDB
4. **檢索比對** → 使用相似度搜尋

### 優勢
- 本地部署，成本低
- 可混合使用 Ollama + MiniMax
- 支援語義標籤分類
