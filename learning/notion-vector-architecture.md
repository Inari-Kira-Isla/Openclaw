## 📊 四個維度優化方案

### 1. 語義分塊 + Context Breadcrumb

**Notion 實現方式**：
- 使用 Callout 區塊標註領域
- 使用 H2/H3 標題進行原子化
- 每個區塊一個獨立頁面或 Block

**範例**：
```
🔵 [領域：海膽鑑定 / 豐洲市場]
## A級馬糞海膽成本分析
...
```

---

### 2. Metadata 影子索引

| 元數據 | Notion 欄位 |
|--------|-------------|
| 角色歸屬 | 角色歸屬 (多選) |
| 時間戳 | 日期 (自動) |
| 領域標籤 | 語義標籤 (多選) |
| 向量狀態 | 向量狀態 (狀態) |

**檢索過濾**：
- 「僅檢索 2026 年且標記為『市場分析』的向量」

---

### 3. 摘要預處理流程

```
Notion 筆記 → LLM 生成摘要 → 寫入「向量摘要」欄位 → ChromaDB 向量化
```

**自動化**：
- n8n 監控 Notion 頁面變動
- 觸發 LLM 生成摘要
- 更新向量資料庫

---

### 4. 定義集資料庫

**Notion 實現**：
- 建立「術語定義」資料庫
- 高頻引用到其他筆記
- 增強向量空間聚類效應
