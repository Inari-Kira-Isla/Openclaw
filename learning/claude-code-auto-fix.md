# Claude Code 自動化修復流程 - 學習筆記

**來源**: Slack 截圖
**日期**: 2026-02-18

---

## 🎯 核心理念

> 從「發現錯誤」→「通知 AI」→「AI 修復」→「發 PR」形成閉環

---

## 🔄 自動化工作流

```
┌─────────────────────────────────────────────────────────────────┐
│            Claude Code 自動化修復流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 錯誤偵測                                                     │
│     監控腳本偵測到 Error/ Critical                               │
│           ↓                                                      │
│  2. Slack 通知                                                  │
│     透過 Webhook 傳到 #alerts 頻道                              │
│           ↓                                                      │
│  3. 人類觸發                                                     │
│     @Claude "幫我調查 修這個 bug"                               │
│           ↓                                                      │
│  4. AI 調查                                                      │
│     Claude Code 讀取程式碼 → 分析根因                            │
│           ↓                                                      │
│  5. 自動修復                                                     │
│     修正程式碼 → 發 PR                                           │
│           ↓                                                      │
│  6. 回報結果                                                     │
│     回 Slack 報告修復內容 + Root Cause Analysis                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 📊 工作階段

### 第一階段：錯誤偵測與告警

| 項目 | 說明 |
|------|------|
| 監控 | 專案中埋入監控腳本 |
| 偵測 | 自動偵測 Error / Critical |
| 告警 | 透過 Webhook 傳到 Slack |

### 第二階段：人機協作觸發 (ChatOps)

| 項目 | 說明 |
|------|------|
| 觸發 | 在 Slack 標記 @Claude |
| 指令 | "幫我調查 修這個 bug" |
| 門檻 | 不需打開 IDE 或登入伺服器 |

### 第三階段：儲存庫上下文關聯

| 項目 | 說明 |
|------|------|
| 識別 | 自動識別錯誤對應的 GitHub Repo |
| 上下文 | 具備跨專案處理能力 |
| 鎖定 | 鎖定特定儲存庫進行作業 |

### 第四階段：自主分析與修復

| 項目 | 說明 |
|------|------|
| 分析 | 閱讀程式碼並模擬故障情境 |
| 修復 | 直接修正程式碼 |
| PR | 自動發送 Pull Request |

### 第五階段：反饋與透明化

| 項目 | 說明 |
|------|------|
| 報告 | 回 Slack 提供總結 |
| RCA | Root Cause Analysis (根本原因分析) |
| 審核 | 人類快速稽核變更 |

## 🔧 實作要素

### 1. Slack 整合

- 建立 Webhook
- 設定 alerts 頻道
- 允許 @Claude 標記

### 2. 監控腳本

- 嵌入專案
- 偵測 Error / Critical
- 觸發 Webhook

### 3. Claude Code

- GitHub 權限
- 發 PR 權限
- 讀寫程式碼能力
