## 摘要

解決 OpenClaw「指令漂移」與溝通摩擦的技術方案：自動化個性化審計 (Personalization Audit)。

**核心問題：** 即便糾正過行為（新對話會重蹈覆轍）
**解決方案：** 自動進化閉環 + 核心覆寫

---

## 詳細內容

### 一、問題痛點

| 痛點 | 說明 |
|------|------|
| 指令漂移 | 糾正後在新對話中重蹈覆轍 |
| 記憶摩擦 | 對話歷史中的糾正會消失 |
| 生產力瓶頸 | 阻礙 Agent 進入生產線 |

### 二、運作邏輯

```
1. 自動掃描
   ↓ 定期回顧對話記錄
2. 摩擦識別
   ↓ 尋找糾正訊號
3. 核心覆寫
   ↓ 更新 USER.md / SOUL.md
4. 本能反應
   ↓ 成為下一個 Session 的行為準則
```

### 三、糾正訊號關鍵詞

- "不對"
- "不是這樣"
- "請修改"
- "我說過"
- "之前說過"

### 四、核心覆寫目標

| 檔案 | 內容 |
|------|------|
| USER.md | 用戶偏好、糾正內容 |
| SOUL.md | 行為準則、溝通方式 |

## 與現有系統整合

### 目前系統缺口

| 功能 | 狀態 | 缺口 |
|------|------|------|
| 每日學習升級 (10:00) | ✅ | 僅分析，無自動修正 |
| 記憶系統 | ✅ | 被動儲存，主動性不足 |
| USER.md / SOUL.md | ✅ | 需手動更新 |

### 整合方案

**優化流程（每日 10:00）：**

```
現有：
1. 收集效能數據
2. 分析對話紀錄
3. 識別問題與瓶頸

優化後：
+ 4. 檢測糾正訊號（"不對"、"不是這樣"）
+ 5. 自動更新 USER.md / SOUL.md
+ 6. 驗證修正效果
```
