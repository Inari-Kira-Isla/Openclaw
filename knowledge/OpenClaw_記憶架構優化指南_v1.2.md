## 詳細內容

### 一、混合動態上下文窗口

將上下文分為三個動態區域：

1. **固定系統指令區 (Anchor)**
   - 放置核心任務邏輯與 Output Schema

2. **動態實體快照區 (Snapshot)**
   - 提取當前任務相關的關鍵變量
   - 例如：魚貨品種、回測參數範圍

3. **滑動對話摘要區 (Rolling Summary)**
   - 僅保留最後 3-5 輪原話
   - 之前的對話轉化為精煉摘要

---

### 二、強化向量檢索 (RAG) 穩定性

**多路召回與重排序 (Reranking)：**
- 通過向量檢索找出前 20 個相關片段
- 使用輕量級重排模型（如 BGE-Reranker）進行二次精選

**評分公式：**
```
Score = α × Sim + β × Recency
```
- Sim：相似度
- Recency：時間衰減因子

**元數據過濾 (Metadata Filtering)：**
- 強制標記 Category（市場數據、代碼邏輯、用戶偏好）
- 檢索時先進行硬過濾，減少張冠李戴

---

### 三、記憶分層存儲架構

| 記憶層級 | 存儲媒介 | 刷新機制 | 性能影響 |
|----------|----------|-----------|-----------|
| L1: 瞬時緩存 | Redis / Memory | 每輪對話更新 | 極高 |
| L2: 任務狀態 | JSON / SQLite | 任務里程碑更新 | 高 |
| L3: 知識長存 | Vector DB (Chroma/Milvus) | 非同步寫入 | 中 |

---

### 四、性能優化設置

1. **本地 Embedding 模型**
   - 部署 BGE-m3 或 Ollama 本地向量模型
   - 減少 API 網絡延遲

2. **異步記憶寫入**
   - 記憶存儲不應阻塞 Agent 思考
   - 將對話總結與向量寫入放入後端異步隊列

3. **上下文壓縮**
   - 開啟 KV Cache 壓縮技術
   - 提升長對話 Token 推理速度

---

### 架構師建議

建立「任務筆記 (Scratchpad)」：
- 在 Prompt 中開闢專屬區域
- 讓 Agent 執行過程中不斷更新進度與關鍵發現
- 比依賴歷史記憶更穩定
