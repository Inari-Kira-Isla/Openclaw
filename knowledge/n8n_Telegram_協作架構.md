# ☁️ n8n + Telegram 協作架構

**版本：** v1.0
**日期：** 2026-02-18
**類型：** 系統架構
**狀態：** 規劃中

---

## 摘要

結合 n8n 自動化 + Telegram 控制，實現分散式 AI 協作系統

---

## 一、架構設計

### 設備分工

| 設備 | 角色 | n8n | Telegram |
|------|------|-----|----------|
| 電腦 A (Alpha) | 決策/策略 | 任務分配 | ✅ 控制 |
| 電腦 B (Beta) | 執行/監控 | RSS/YouTube | - |
| 雲端 | 數據同步 | ✅ 協調 | ✅ 訊息 |

### 數據流向

```
Telegram 用戶
     │
     ▼
┌────────────────┐
│  n8n Webhook  │ ← 接收指令
└───────┬────────┘
        │
   ┌────┴────┐
   ▼         ▼
電腦 A   電腦 B
(執行)   (執行)
   │         │
   └────┬────┘
        ▼
   數據庫同步
```

---

## 二、n8n 工作流

### 1. 指令接收工作流
- 觸發：Telegram Webhook
- 功能：接收指令 → 解析 → 分發

### 2. 任務分配工作流
- 觸發：定時 / Webhook
- 功能：根據設備負載分配任務

### 3. 數據同步工作流
- 觸發：定時 (每小時)
- 功能：同步 Notion / 數據庫

### 4. 監控工作流
- 觸發：定時 (每 6 小時)
- 功能：RSS / YouTube 監控

---

## 三、Telegram 控制

### 指令列表

| 指令 | 功能 |
|------|------|
| `/status` | 查看系統狀態 |
| `/tasks` | 查看任務列表 |
| `/execute [任務]` | 執行任務 |
| `/sync` | 手動同步數據 |
| `/logs` | 查看日誌 |
