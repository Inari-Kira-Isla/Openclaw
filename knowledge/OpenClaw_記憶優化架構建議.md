# 🧠 OpenClaw 記憶優化架構建議

**版本：** v1.1
**日期：** 2026-02-17
**類型：** 系統架構
**標籤：** Agent, 架構, 技能
**狀態：** 已完成

---

## 摘要

AI 系統架構師提供 OpenClaw 記憶優化方案：
- 三層記憶模型：短期/工作/長期
- 遞歸摘要、語義壓縮、遺忘曲線權重
- 技術棧：Redis + PostgreSQL (pgvector)

---

## 一、多層級記憶模型

### 1. 短期記憶 (Short-term Memory)
- **對象：** 當前對話的原始 Message 序列
- **優化：** 滑動窗口 (Sliding Window) 機制，保留最近 N 輪對話
- **目的：** 確保反應速度、控制 Token 消耗

### 2. 感官/工作記憶 (Working Memory)
- **對象：** 當前任務的核心變量、用戶偏好、實體信息
- **優化：** 提取關鍵事實，以結構化 JSON 形式存儲
- **應用：** 掛載在 System Prompt 中

### 3. 長期記憶 (Long-term Memory)
- **對象：** 歷史對話摘要、領域知識庫
- **優化：** 整合向量數據庫 (Vector DB)
- **檢索：** 語義搜尋

---

## 二、核心優化策略

### A. 遞歸摘要
- **觸發條件：** 對話長度超過閾值
- **機制：** 舊對話 + 原有摘要 → 生成新摘要
- **好處：** 壓縮 Token 佔用，保持上下文邏輯

### B. 語義壓縮與實體索引
- 識別對話中的實體（人名、公司、產品）
- 建立知識圖譜片段
- 優先匹配相關實體，避免向量檢索噪音

### C. 遺忘曲線與權重管理
- **機制：** Importance (重要度) + Recency (新鮮度) 權重
- **公式：** Score = Importance × Recency^{decay_factor}
- **應用：** 定期清理低分記憶，保持精煉性

---

## 三、架構整合方案

| 組件 | 技術棧 | 功能 |
|------|--------|------|
| 緩存層 | Redis | Session 級短期記憶，毫秒級讀取 |
| 存儲層 | PostgreSQL (pgvector) | 歷史對話向量化，支援語義檢索 |
| 調度器 | OpenClaw Controller | 判斷 RAG 或觸發摘要邏輯 |

---

## 四、優先建議

針對 Retrolyze (回測工具) 或澳門企業 AI 方案，建議優先實現「結構化工作記憶」：
- 記住用戶的特定業務邏輯（如海膽定價模型）
- 不需要從海量歷史對話中檢索
