# 簡單重複工作模式 - 2026-03-05

## 定時任務類型

### 1. 訓練數據收集 (Training Data Collection)
- **頻率**: 每日
- **觸發**: scheduled_reminder
- **職責**: 
  - 記錄簡單重複工作
  - 生成訓練範例
  - 建立訓練數據集
  - 儲存到 training-data/

### 2. 系統健康檢查 (System Health Check)
- **頻率**: 每小時/每日
- **檢查項目**:
  - Gateway 狀態
  - Sessions 數量
  - Context 使用率
  - 記憶體使用

### 3. 錯誤學習 (Error Learning)
- **頻率**: 每小時
- **職責**:
  - 檢查新錯誤日誌
  - 分析錯誤模式
  - 記錄學習要點

### 4. 記憶體監控 (Memory Monitoring)
- **頻率**: 每日多次
- **職責**:
  - 監控記憶體變化
  - 分析新檔案
  - 更新記憶庫

### 5. Token 監控 (Token Monitoring)
- **頻率**: 每日/定時
- **職責**:
  - 追蹤 token 使用量
  - 分析成本趨勢
  - 記錄異常閾值

---

## 重複工作識別要點

1. **定時性**: 是否有固定執行時間
2. **結構性**: 是否有固定輸入/輸出格式
3. **可自動化**: 是否可腳本化處理
4. **訓練價值**: 是否可作為訓練範例

## 收集原則

- 每次執行後記錄為 JSON
- 包含時間戳與任務類別
- 記錄關鍵步驟與輸出
- 分析模式以便優化
