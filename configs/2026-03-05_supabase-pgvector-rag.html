Supabase + pgvector：向量資料庫配置與 RAG 系統搭建

📅 2026-03-05 ⏱ 8 分鐘閱讀 ✍️ AI 學習寶庫

Supabase + pgvector 向量資料庫配置與 RAG 系統搭建完整教學

Supabase 結合 pgvector 擴展功能，讓您能夠在熟悉的 PostgreSQL 環境中儲存與搜尋向量資料，最適合用於建構檢索增強生成（RAG）系統。本文將詳細說明如何配置向量資料庫並實作語義搜尋功能。

啟用 pgvector 擴展

首先需要在 Supabase 專案中啟用 pgvector 擴展。在 Supabase 儀表板的 SQL Editor 中執行以下指令：

-- 啟用 pgvector 擴展
CREATE EXTENSION IF NOT EXISTS vector;

-- 建立向量資料表
CREATE TABLE document_chunks (
  id BIGSERIAL PRIMARY KEY,
  content TEXT NOT NULL,
  embedding vector(1536),  -- OpenAI text-embedding-ada-002 維度
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 建立 HNSW 索引加速相似性搜尋
CREATE INDEX ON document_chunks 
USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);

pgvector 提供三種距離運算方式：內積（inner_product）、餘弦相似度（cosine）、歐氏距離（euclidean）。上述範例使用餘弦相似度，適合文字嵌入比對。

儲存與檢索向量資料

將文字轉換為向量後即可存入資料庫。以下是使用 Python 的基本操作流程：

# 安裝必要套件
pip install supabase openai

from supabase import create_client
import openai

# 初始化客戶端
supabase = create_client(SUPABASE_URL, SUPABASE_KEY)
openai.api_key = OPENAI_API_KEY

# 產生文字嵌入
def get_embedding(text):
    response = openai.Embedding.create(
        model="text-embedding-ada-002",
        input=text
    )
    return response['data'][0]['embedding']

# 儲存文件向量
def store_document(content, metadata={}):
    embedding = get_embedding(content)
    supabase.table('document_chunks').insert({
        'content': content,
        'embedding': embedding,
        'metadata': metadata
    }).execute()

# 語義搜尋
def semantic_search(query, top_k=5):
    query_embedding = get_embedding(query)
    result = supabase.rpc(
        'match_documents',
        {
            'query_embedding': query_embedding,
            'match_threshold': 0.7,
            'match_count': top_k
        }
    ).execute()
    return result.data

建立相似性搜尋函數

在 PostgreSQL 中建立 RPC 函數以執行向量相似性搜尋：

CREATE OR REPLACE FUNCTION match_documents(
  query_embedding vector(1536),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE (
  id BIGINT,
  content TEXT,
  metadata JSONB,
  similarity FLOAT
)
LANGUAGE plpgsql
AS $$
BEGIN
  RETURN QUERY
  SELECT
    dc.id,
    dc.content,
    dc.metadata,
    1 - (dc.embedding <=> query_embedding) AS similarity
  FROM document_chunks dc
  WHERE 1 - (dc.embedding <=> query_embedding) > match_threshold
  ORDER BY dc.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;

此函數使用餘弦距離運算子（<=>）計算相似度，回傳符合閾值的最高相似度文件區塊。

整合 RAG 系統架構

RAG 系統的核心流程包含以下步驟：首先將外部文件分塊並轉換為向量儲存；接著當用戶提出問題時，將問題轉為向量進行相似性搜尋；最後將檢索到的相關內容與問題一同送入語言模型生成回答。

# RAG 完整流程範例
def rag_query(user_question):
    # 1. 語義搜尋相關文件
    relevant_docs = semantic_search(user_question, top_k=3)
    
    # 2. 建立上下文
    context = "\n\n".join([doc['content'] for doc in relevant_docs])
    
    # 3. 生成回答
    prompt = f"""根據以下參考資料回答問題。如果資料不足，請據實告知。

參考資料：
{context}

問題：{user_question}

回答："""
    
    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}]
    )
    
    return response['choices'][0]['message']['content']

效能優化建議

  • 選擇合適的索引類型：HNSW 適合需要高速搜尋且準確度要求高的場景；IVFFlat則在資料量較小時更節省資源
  • 批次處理：大量文件向量化和儲存時使用批次操作，減少資料庫連線開銷
  • 過濾條件：在向量搜尋時結合 SQL 過濾條件，如時間範圍或特定分類
  • 快取策略：對常見問題的搜尋結果進行快取，降低向量計算頻率

總結

Supabase + pgvector 提供了一套完整且易於使用的向量資料庫解決方案，讓開發者能夠在熟悉的 PostgreSQL 環境中快速建立 RAG 系統。透過本文的教學，您應該能夠獨立完成向量資料表設計、儲存檢索功能實作，以及與語言模型的整合工作。

常見問題

pgvector 支援哪些向量維度？

pgvector 支援最高 16,000 維的向量，實際維度需根據使用的嵌入模型決定，如 OpenAI ada-002 為 1536 維。

HNSW 索引需要多少記憶體？

HNSW 索引記憶體占用約為資料量的 10-20%，可透過調整 m 和 ef_construction 參數在效能與記憶體間取得平衡。

如何提升 RAG 系統的準確度？

建議調整文件分塊大小、優化嵌入模型選擇、增加中繼資料過濾，並使用 rerank 模型對搜尋結果重新排序。
